之前，笔者复现了曹春晓老师提出的耀眼波动率因子。

当时，并没有看到研报的原文，只是看到了这个因子。这次，笔者找到了研报的原文，发现还有一个叫做耀眼收益率的因子。于是，想看看这个耀眼收益率因子是不是也和耀眼波动率一样出色！

由于两个因子都是出自同一篇研报《成交量激增时刻蕴含的alpha信息》。所以，在计算上是有相似的地方的。

这里主要介绍一下两者的区别。

耀眼波动率计算的是成交量激增时刻n到n+4的波动率，一天有n个成交量激增时刻就有n个波动率，然后对这些波动率求均值就得到了日耀眼波动率。

耀眼收益率就没那么复杂了，只需要对所有成交量激增时刻的收益率求均值即可，具体实现如下。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'close'])
    ret = data.to_dataframe('close')
    ret = ret.pct_change().iloc[1:-1]
    # ret = ret.rolling(5).std()

    vol = data.to_dataframe('volume').iloc[1:-1]
    vol = vol.diff()
    thd = vol.mean(axis=0) + vol.std(axis=0)
    vol = (vol > thd).replace(False, np.nan)
    dazzling = (vol * ret.reindex(data.codes, axis=1)).mean()
    dazzling.name = 'dazzling'
    fill_dazzling = np.abs(dazzling - dazzling.mean())
    fill_dazzling.name = 'fill_dazzling'
    res = pd.concat([dazzling, fill_dazzling], axis=1)
    res.index.name = 'code'
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    return res.reset_index()
```

代码方面主要的改动是两点。

第一，第10行代码被注释掉了，这是原来计算波动率的时候需要的。

第二，第16行代码，原来对vol进行了shift，现在也去掉了。因为只需要计算成交量激增时刻的收益率，原来则是成交量激增时刻n到n+4的波动率。

总结：这个因子的在IC上的表现依旧出色，但是分层回测的表现却相对一般。因子值最大的一组和因子值最小的一组在2018年到2024年这个区间里的年化收益率都是低于其他三组的。

总体来说，这个因子不如耀眼波动率因子。虽然，它在2018年的IC绝对值也超过了0.08，比耀眼波动率因子高，但是它月度IC出现反向的情况也比耀眼波动率因子要多，因此这个因子的稳定性不如耀眼波动率。同时，从分层回测结果来看，即使作为空头因子，它的表现也不如耀眼波动率。因为，它因子值最大的一组年化收益率是高于耀眼波动率因子值最大的一组的。
