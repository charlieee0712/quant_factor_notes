本文，我们继续来复现曹春晓老师的因子。

该因子是在方正证券研究所于2024年3月19日发布的报告《日内协同股票性价比度量与“协同效应”因子构建——多因子选股系列研究之十六》中提到的。

这篇研报中提到了两个因子，但是这两个因子的计算差异很大，所以笔者决定分成两篇文章来介绍。

本文先介绍成交量协同因子，下一篇文章将介绍协同价差因子。从研报的结果来看，第二个因子的效果可能会非常好，IC绝对值超过了0.1。但是，笔者在写这篇文章的时候，协同效应因子还没有计算完，所以效果只能等明天了。如果大家收不到推送的话，可以将公众号设置以下星标，这样就能第一时间看到文章了。同时，没有点关注的朋友们能不能帮忙点点关注，谢谢大家。

好了，回到成交量协同因子，先介绍因子的计算步骤，然后结合代码进一步说明。

第一步，在t分钟末，取t-4分钟到t分钟的最高价、最低价、收盘价和开盘价一共20分数据，计算其均值和标准差。比较t分钟收盘价与均值和标准差的大小，将个股划分成三个区间，大于上轨，低于下轨和两者之间。将于该标的处于相同区间的其他标的定义为协同效益标的。

其中，上轨=均值+标准差；下轨=均值-标准差。

第二步，计算标的t分钟的成交量占比和协同标的t分钟成交量占比之和。

第三步，计算标的成交量占比和协同标的成交量占比之和的相关系数。若股票自身成交量与“协同成交量”间的相关系数越大，说明该股票更能代表协同股票中的成交量变化特征，从而使得该股票在协同股票中的交易辨识度更高，更容易受到投资者的关注。（研报原文）

具体代码如下：

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['high', 'low', 'close', 'open', 'volume'])

    vol = data.to_dataframe('volume')
    vol = vol.div(vol.sum(axis=0), axis=1).iloc[5:]
    res = Parallel(n_jobs=-1, verbose=10)(
        delayed(self.cal_factors)(data.data[:, :, idx])
        for idx in range(len(data.codes))
    )
    res = pd.DataFrame(data=np.array(res).T, columns=data.codes, index=data.trade_days[5:])
    for key in ['u', 'l', 'b']:
        tmp = ((res == key) * vol).sum(axis=1)
        res[res == key] = np.tile(tmp.values[:, None], (1, len(res.columns)))
    res = res - vol
    flag = vol.columns[vol.std() == 0].tolist()
    vol[flag] = np.nan
    df = vol.corrwith(res, method='pearson')
    df.name =  pd.to_datetime(date_str) + timedelta(hours=15)
    return df
```

代码分为两部分，关于cal\_factors方法，将在后面介绍。

前7行，加载数据。

第8-9行，计算每个标的每分钟成交量占比，去掉前4分钟是因为计算上下轨的时候是从第五分钟开始的。

第11-15行，计算每个标的每分钟所处的区间，用'u'表示大于上轨，'l'表示低于下轨，'b'表示两者中间。

第16-19行，计算每一个标的的协同成交量。其中，第19行是去掉自己的成交量占比。

第20-21行，将vol的标准差为0的标的对应的成交量置为nan，否则在用corrwith的时候会报错。

```python
@staticmethod
def cal_factors(data):
    res = []
    for i in range(5, len(data)):
        all_price = data[i-5:i, :]
        sigma = np.nanstd(all_price)
        mu = np.nanmean(all_price)
        if data[i-1, 2] > sigma + mu:
            res.append('u')
        elif data[i-1, 2] < mu - sigma:
            res.append('l')
        else:
            res.append('b')
    return res
```

这一部分代码很简单，就是纯粹的遍历然后判断标的所处的位置。
