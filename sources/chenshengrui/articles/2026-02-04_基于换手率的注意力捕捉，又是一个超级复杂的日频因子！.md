**基于换手率的注意力捕捉**

这个因子是基于异常收益率的注意力捕捉因子的姊妹篇，笔者在[基于异常收益率的注意力捕捉，一个结合了涨跌停数量、Fama-French三因子模型和动量的超复杂因子！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247494289&idx=1&sn=ac78d20e2d7ae806dd70c3ecea64ed73&scene=21#wechat_redirect)这篇文章中复现过。

一般看到将因子名字中的收益率换成换手率之后，第一时间想到的就是将所有的收益率都换成换手率来计算因子，这是一种很常见的做法。

但是，这个基于换手率的注意力捕捉因子不是这样的。它只是将基于异常收益率的注意力捕捉因子时序回归中的解释变量（行业涨跌停占比/全市场涨跌停占比）换成了换手率，其他的不变。

关于这个因子，其背后的逻辑，2025年的《因子日历》上是这样描述的。

> 不同股票对同一市场关注度事件的反映程度往往是不同的，可借助个股对市场关注度事件的敏感程度来衡量股票对该事件的注意力捕捉效应；捕捉能力越强（敏感程度越高），表明个股更能吸引市场注意，从而导致买盘压力增大，股价高估；该因子以换手率作为市场关注度事件，换手率衡量投资者交易热度，反映关注水平。

**计算步骤和代码**

理论上，计算步骤没什么复杂的了，甚至不值得拿出来说了。

但是，笔者没看到研报原文，只是在2025年的《因子日历》上看到了这个因子。

在《因子日历》上，文字描述的是行业中所有标的的平均换手率，而公式确实行业中所有标的换手率的标准差。

所以，笔者在实现的时候计算了两个因子，一个是基于换手率的均值的，另一个则是基于换手率的波动的。

**1**

**代码**

关于代码部分，只是将原来的cal\_limit\_num方法换成了cal\_tr\_data。

```python
@staticmethod
    def cal_tr_data(group):
        group = group.dropna()
        group['tr_mu'] = group['tr'].mean()
        group['tr_sigma'] = group['tr'].std()
        return group[['tr_mu', 'tr_sigma']]
```

其他的就是在加载数据的时候去掉了一些冗余的字段，精简了一下代码。

**因子评价**

![](images/2026-02-04_基于换手率的注意力捕捉，又是一个超级复杂的日频因子！/002.png)

用换手率的均值和标准差计算出来的因子，它们之间的相关性并不高，没有超过0.05。

从IC上来看，这两个因子也是差不多的，但是分层回测上用换手率均值计算的因子表现稍微好了那么一点点，所以下面展示的因子评价结果是基于换手率均值计算的。

同时，由于这个因子在计算时序回归的时候已经使用了过去21个交易日的数据了，所以不再需要用均值或者标准差来合成月度因子了。

当然，要合成，也是可以的，就相当于两次平滑了。只不过因子的表现就不一定会发生什么变化了，至少在这个因子中再用过去21个交易日的均值或者标准差合成一次的话，效果是会变差的。
