**计算公式**

![](images/2025-11-26_AR价差，死了一个脑细胞后，可以整出来快242个因子？还有因子连续六年IC绝对值超0.1！/001.png)

公式可以说是非常简单。

但是有两个问题。

第一个问题，E的话通常是求期望的意思，那么到底是用多长时间的数据求期望呢？笔者不知道，于是尝试了一下1、5、15、30、60和240。240分钟的数据求期望，每天就一个值。如果是其他分钟的数据求期望，每天还有多个值，然后得再求一次期望得到日频的因子。

第二个问题，这里的价格要不要取对数。

如果不取对数，那么有这样一个问题。

假设一个标的的，收盘价是100，最高价是105，最低价是95。另一个标的的收盘价是10，最高价是10.5，最低价是9.5。在不取对数的情况下，这两个标的的因子值是不一样的。换句话说，这个因子的因子值大小和价格有关。这时候，复不复权对最终的因子值具有颠覆性影响，甚至前复权和后复权得到的结果都不一样。

所以，笔者在计算的时候对收盘价和（最高价+最低价）/2这两个价格取了对数。

**2**

**代码**

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    cur = pd.to_datetime(date_str) + timedelta(hours=15)
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['high', 'low', 'close'])
    high = data.to_dataframe('high')
    low = data.to_dataframe('low')
    close = np.log(data.to_dataframe('close').shift(1))
    mid = np.log((high + low) / 2)
    data = (close - mid.shift(1)) * (close - mid)
    factor1 = np.sqrt(((-4 * data).clip(lower=0)).mean())
    factor2 = np.sqrt((-4 * data.mean()).clip(lower=0))
    res = [factor1, factor2]
    cols = []
    for i in [5, 15, 30, 60]:
        res.append(np.sqrt((-4 * data.rolling(i).mean()).clip(lower=0).mean()))
        cols.append('hf_ar_{}'.format(i))
    res = pd.concat(res, axis=1)
    res.columns = ['hf_ar', 'hf_ar_240'] + cols
    res['datetime'] = cur
    return res
```

在上面那两个问题浪费了笔者一个脑细胞之后，写出了这样的代码。

前6行，读取数据。

第7-10行，获取价格数据，并对收盘价和中间价（最低价和最高价的均值）取对数。

第12-19行，就是按照不同的时间长度计算因子值，具体参考第一个问题。其实，这里可以从1到240的时间窗口都计算一个的，但是没有必要，因为相关性会很高。

同时，这个因子也可以用日频数据来计算，具体思想和分钟是一样的，这里不赘述了。笔者对日频数据计算了1日的AR价差然后取21日均值（ar\_daily，对应1分钟的版本），还计算了21日AR价差因子（ar\_mean，对应240分钟的版本）

从相关性来看，hf\_ar到hf\_ar\_240，相关性逐渐降低。

而两个基于日频数据计算的因子，和其他因子的相关性普遍都比较低，这一点和之前的两个价差因子是一致的。

先说日频数据计算的，这两个因子IC绝对值低，分层回测也比hf的差。所以，就不看它们的因子评价结果了。

对于hf类的数据，除了hf\_ar\_240外，其他的差距都在毫厘之间。同时，笔者尝试了用过去21个交易日的均值或者标准差来合成因子，两者的IC绝对值是差不多的，分层回测也是类似。

在笔者综合对比了之后，个人认为用过去21个交易日的均值合成hf\_ar\_30是表现最好的，所以下面展示的是这个因子的因子评价结果。
