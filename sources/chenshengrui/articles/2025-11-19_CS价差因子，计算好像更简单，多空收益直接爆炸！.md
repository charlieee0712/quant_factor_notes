**CS价差因子**

在[ROLL价差因子，八种价差因子中计算最简单的，效果会怎么样？](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247491751&idx=1&sn=97d36d3674a00bae3b501117a561ff78&scene=21#wechat_redirect)这篇文章中，笔者复现了第一个价差因子，也是最简单的价差因子。

本文，笔者将复现第二个，CS价差因子。它的优点是相较于ROLL价差估计更精确，缺点是计算过程较复杂，同时在交易不频繁的时候CS模型可能无法准确估计买卖价差。

**计算公式和代码**

对于，缺点中的计算过程比较复杂，笔者不敢苟同。

因为，从代码实现的角度来看，这个因子的计算可以说是非常简单了，甚至能够自如的在分钟频数据和日频数据上进行切换。

**计算公式**

![](images/2025-11-19_CS价差因子，计算好像更简单，多空收益直接爆炸！/001.png)

这个公式是基于日频数据的。

如果换成分钟级别的数据的话，只需要再进行一个日内求均值的操作就可以了。

**代码**

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['high', 'low'])
    high = data.to_dataframe('high')
    low = data.to_dataframe('low')
    beta = np.power(np.log(high / low), 2).rolling(2).mean()
    gamma = np.power(np.log(high.rolling(2).max() / low.rolling(2).min()), 2)
    num = 3 - 2 * np.sqrt(2)
    alpha = (np.sqrt(2 * beta) - np.sqrt(beta)) / num - np.sqrt(gamma / num)
    res = (2 * np.exp(alpha - 1)) / (1 + np.exp(alpha))
    res = res.mean()
    res.name = pd.to_datetime(date_str) + timedelta(hours=15)
    return res
```

前7行，读取数据。

第8行，计算beta。

第9行，计算gamma。

第10行，计算alpha计算公式中的分母。因为这个值需要计算两次，所以这里用了一个变量保存，减少重复计算。

第11行，计算alpha。

第12行，计算因子值。

说实话，这个代码真的是非常简单了。所以，笔者对于它计算复杂度高的这个说法不敢苟同。同时，这个因子最复杂的运算也就是指数运算了，和ROLL价差的协方差比起来应该算是简单的。

**因子评价**

![](images/2025-11-19_CS价差因子，计算好像更简单，多空收益直接爆炸！/003.png)

这里，先来看看分钟频数据计算的因子和日频数据计算的因子的相关性。看起来还行，相关性低于0.5了。

值得一提的是，这里的日频数据只用了两天的最高价和最低价来计算。因此，在进行21个交易日调仓的时候，依旧可以用过去21个交易日的均值或者标准差来合成“月度因子”。

这里，分钟数据和日频数据有一点差别。

日频数据，用均值合成的IC绝对值高于用标准差合成的，但两者分层回测的表现差不多。而分钟数据，则是用标准差合成的在各方面都优于均值合成的。同时，分钟数据用均值合成的效果和日频数据的均值合成的效果是类似的。
