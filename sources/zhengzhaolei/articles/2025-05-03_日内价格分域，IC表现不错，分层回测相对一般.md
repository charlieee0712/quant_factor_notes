上一篇文章介绍了兴证金工2024年7月31日发布的研报《高频研究系列九——日内分域信息中的Alpha》中提到的日内时间分域因子。

本文将介绍一下日内价格分域因子。

与时间分域因子相比，价格分域因子的计算就相对复杂了一点。因为这个因子需要找到日内最高点和最低点，然后在这个基础上计算三个因子。

最高点的计算，对标的的日内最高价按照百分比排序，然后每个时间点计算过去15分钟排序的均值，最后排序均值最大的点就是日内高价时点。

最低点的计算，对标的的日内最低价按照百分比排序，然后每个时间点计算过去15分钟排序的均值，最后排序均值最小的点就是日内低价时点。

有了高价时点和低价时点知乎，就能计算三个因子了。

第一个，diff\_idx，高价时点和低价时点的分钟差。这里有两种计算方法，按照交易分钟计算，也可以按照实际的分钟计算。但是笔者尝试了一下，无论哪个效果都不好。

第二个，diff\_std，高价时点和低价时点的波动差。一个点肯定是没办法计算波动的，这里的波动是指时点前15分钟（包括时点）的波动率。

第三个，diff\_vol，高价时点和低价时点的成交量占比差。这里计算的也是过去15分钟成交量占比之和。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close', 'open', 'volume', 'high', 'low'])
    dfs = data.to_dataframes()
    ret = dfs['close'] / dfs['open'] - 1

    high_idx = (dfs['high'].rank(pct=True, axis=0)).rolling(15).mean().idxmax().iloc[1:]
    low_idx = (dfs['low'].rank(pct=True, axis=0)).rolling(15).mean().idxmin().iloc[1:]
    diff_idx = high_idx - low_idx

    sigma = ret.rolling(15).std()
    vol = dfs['volume'].div(dfs['volume'].sum(axis=0), axis=1)
    vol = vol.rolling(15).sum()
    sigma = (pd.Series({col: sigma.loc[row, col] for col, row in high_idx.items()}) -
             pd.Series({col: sigma.loc[row, col] for col, row in low_idx.items()}))
    vol = (pd.Series({col: vol.loc[row, col] for col, row in high_idx.items()}) -
             pd.Series({col: vol.loc[row, col] for col, row in low_idx.items()}))
    res = pd.concat([diff_idx, sigma, vol], axis=1)
    res.columns = ['diff_idx', 'diff_std', 'diff_vol']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```
