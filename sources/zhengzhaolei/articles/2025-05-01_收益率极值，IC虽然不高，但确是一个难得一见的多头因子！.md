本文，要复现的因子来自兴证金工2022年5月4日发布的研报《高频研究系列三——收益率分布中的 Alpha(2)》，作者郑兆磊。

在之前的文章中，笔者复现了其中的一部分因子了，就是基于高斯混合分布然后将分钟收益率划分为跳价期和震荡期。

本文要介绍的是一基于收益率极值的因子。

这一系列因子的复现主要是基于猜测，因为在研报中关于极值就提到了一句我们参考VaR 与 Expected Shortfall（ES）对于收益极端值的刻画思路，估计极值exRtn 因子值。研报中没有具体的计算公式和步骤，所以笔者在向AI请教了之后，随便尝试了一下。

然后，在这一思路下，研报给出了两个因子。

第一个，exRtn\_maxVal，5分钟滚动收益率大于VaR的样本均值/VaR。

第二个，exRtn\_minFre，5分钟滚动收益率小于VaR的样本个数。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close', 'open'])
    dfs = data.to_dataframes()
    ret = dfs['close'].pct_change(1)
    thd = ret.quantile([0.1, 0.9]).values
    up = ret > thd[1:, :]
    low = ret < thd[0:1, :]
    max_freq = up.mean(axis=0)
    min_freq = low.mean(axis=0)
    max_val = (ret * np.where(up, 1, np.nan)).mean(axis=0) / thd[1]
    min_val = (ret * np.where(low, 1, np.nan)).mean(axis=0) / thd[0]
    res = pd.concat([max_freq, min_freq, max_val, min_val], axis=1)
    res.columns = ['max_freq', 'min_freq', 'max_val', 'min_val']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

核心计算代码从第9行开始，这一行计算了一分钟收益率。

第10行，VaR门限。这里需要注意的是，门限设置的是0.1和0.9，因为在测试的时候效果比较好。

第11行，计算大于VaR的情况。

第12行，计算小于VaR的情况。

第13-14行，计算两个概率。一个是大于VaR的，一个是小于VaR的。

第15-16行，计算两个值，max\_val在前面提到过，min\_val就是收益率小于VaR的样本均值/VaR。

关于这一系列因子，研报中是这样解释的极大值相关因子（极大值的幅度和频率）的因子值越大，

预期收益越低，而极小值相关因子的因子值越大，预期收益越高。

这里提到的5分钟滚动收益率按照字面理解和5分钟收益率应该是不一样的，但是在复现过程中笔者发现直接用1分钟的数据效果更好，因此就直接展示1分钟数据的结果了。

总结：虽然这一系列因子的IC不是很高，但是从分层回测的结果来看，因子值最小的一组收益率居然遥遥领先其他四组。如此看来，确实是一个难得一见的多头因子，毕竟在之前的研报复现中，绝大多数都是空头因子。

值得一提的是，这两个因子都是用均值合成的，用标准差合成的话会出现IC一些年份为正一些年份为负这样的不稳定的情况。至于两个和频率有关的因子，无论是用均值合成还是用标准差合成，都不是一个好因子。这也是显而易见的，因为VaR的门限是通过百分比来选择的，这也就意味着所有标的在这两个因子上的取值应该是相等的，它们的差异最多来自于中性化。

所以，看起来研报中的VaR应该不是这个意思了。

同时，笔者也尝试过用均值+3倍标准差的方式，效果也很一般。

在收益率服从正态分布的假设中，笔者进行了多次尝试，效果最好的是假设收益率是均值为0的正态分布，然后计算方差， 用3倍方差作为收益率极值判断的门限。但是，这个方法从数学上不太说得通，所以也就放弃了。
