本文的因子收到了兴证金工的首席分析师，郑兆磊来的在2022年5月4日发布的研报《高频研究系列三——收益率分布中的Alpha（2）》的启发。

或者说，完全来自于这个研报，这是在这个研报中并没有介绍这个因子。

![](images/2025-03-25_跳价期波动，原理很复杂，但计算并不难，2023年IC绝对值超过0.1，2021年后半段的分层回测也不错/001.jpg)这个因子的原理很复杂，设计到了高斯混合分布，以及估计高斯混合分布参数的期望最大（EM）算法。

简单来说，一般情况下，估计分布的参数使用最大似然估计的。但是对于高斯混合分布来说，由于它是多个高斯分布相加得到的，因此最大似然估计并不适用。因为，对数的加法是一个很难处理的问题。

具体的细节，大家可以自从查阅资料去了解。

虽然这个因子的计算很复杂，但是得益于开源包的广泛应用，实现起来却一点而也不复杂。这一块，笔者将在代码说明里面详细介绍一下。

继续回到原理层面。

研报中引用了1973年的一篇论文，这篇论文用高斯混合分布来研究日度收益率的分布，而这篇研报将其应用到了日内。假设一个标的的日内收益率是由两个高斯分布混合而成的，权重大的均值约等于0的那个高斯分布对应的时期称为震荡期，权重小均值不等于0的那个高斯分布对应的时期称为跳价期。

然后取跳价期对应的高斯分布的均值作为因子。

研报中，对于这个因子的背后的逻辑也有解释。简单来说，跳价期代表了大额交易者的交易动向，如果大额交易者推动股价下跌，那么当前股价低于公允价格，未来将上涨。反之，如果大额交易者推动股价上涨，那么当前股价高于公允价格，未来价格将下跌。

所以，这是一个IC为负的因子。

但是，在笔者实测中，这个因子的效果并不是最好的。于是根据研报中提到的内容，整出了跳价期的波动这个因子。虽然，在研报中没有直接介绍这个因子，但笔者还是觉得这个因子应该属于这个研报。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close'])
    ret = data.data[1:, 0, :] / data.data[:-1, 0, :] - 1
    ret = ret[:-1, :]
    res = []
    for i in range(len(data.codes)):
        res.append(self.gmm_fit(ret[:, i]))
    res = pd.Series(data=res, index=data.codes, name=pd.to_datetime(date_str) + timedelta(hours=15))
```

第8-9行，计算日内收益率，这里我们的日内收益率使用收盘价计算的。同时去掉了第一根和最后一根k线。理论上，也可以通过收盘价和开盘价来计算，感觉差距不会太大，感兴趣的读者可以自行尝试一下。

第11-12行，通过循环遍历每一个标的，执行gmm\_fit方法来求跳价期均值。

```python
@staticmethod
def gmm_fit(data):
    gmm = GaussianMixture(n_components=2, covariance_type='full')
    gmm.fit(data.reshape(-1, 1))
    keys = gmm.weights_
    mu = gmm.means_.reshape(-1)
    sigma = np.sqrt(gmm.covariances_).reshape(-1)
    if keys[1] > keys[0]:
        return sigma[1]
    return sigma[0]
```

第3行，通过 sklearn的GaussianMixture方法来拟合高斯混合模型，n\_components设置高斯混合分布的分量为2。

第4行，拟合高斯混合分布。

第5行，获取权重。

第6行，获取均值。

第7行，获取方差。

由于高斯混合分布的估计精度和序列长度是有关系的，所以在因子评价的时候只展现1分钟和5分钟数据的结果，频率再低的话，很难准确估计高斯混合模型的参数。
