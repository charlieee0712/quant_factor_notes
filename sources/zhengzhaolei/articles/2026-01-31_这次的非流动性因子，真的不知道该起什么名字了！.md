**非流动性**

![](images/2026-01-31_这次的非流动性因子，真的不知道该起什么名字了！/001.jpg)自从Amihud提出来非流动性因子之后，这个世界上到底诞生了多少非流动性因子，笔者不得而知。但是，从笔者复现的记录来看，已经两只手都数不过来了。

可是，关于非流动性因子的研究被没有止步于此，笔者发现兴业证券的郑兆磊老师在2025年11月18日发布的研报《高频系列十一——流动性因子全解析：选股、择时与多策略》。

在这篇研报中提到的和非流动性相关的因子非常多，简单来说可以分成三类。本文，笔者和大家介绍的是第一类，该因子考虑了将每只股票和市场整体流动性进行比较。

**计算步骤和代码**

**该因子通过引入表征个股流动性跟随市场流动性变化的Beta系数指标，能够更好反映其在选股截面上的流动性相对情况。**

为什么这样做呢？

是因为Amihud是一个仅依赖单只股票数据计算得到的流动性因子，类似于物理学概念中的绝对速度和相对速度，其绝对数值在一定程度上缺乏好可比性。

**1**

**计算步骤**

以标的在t时间相对于t-1时间的变化率为被解释变量，市场在t时间的流动性变化率为解释变量进行时序回归，取回归的beta作为最后的因子。

关于时间窗口，如果是1分钟的数据，则每天单独回归，然后过过去21天取均值。如果是5分钟或5分钟以上的数据，则将过去21天的数据一起进行时序回归，得到最后的因子。

关于变化率的计算，研报中使用的是对数差分。

关于市场流动性变化率，研报中取的是截面所有标的的均值。

**2**

**代码**

由于笔者在测试的时候发现，1分钟的数据表现更好，所以代码这一块也就只展示一下用1分钟数据计算的了。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    if idx < 243 or pd.to_datetime(date_str) >= pd.to_datetime('2026-01-01'):
        return pd.DataFrame({})
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'close', 'open', 'turnover'])
    rtn = data.to_dataframe('close') / data.to_dataframe('open') - 1
    vol = data.to_dataframe('volume')
    money = data.to_dataframe('turnover')
    data.to_dataframe('volume')
    tmp_cap = self.cap.iloc[idx - 243].reindex(data.codes)
    tr = vol / tmp_cap.values.reshape(1, -1)
    vol = (np.log(rtn.abs() / vol)).diff()
    money = (np.log(rtn.abs() / money)).diff()
    tr = (np.log(rtn.abs() / tr)).diff()
    res = []
    for data in [vol, money, tr]:
        res.append(self.cal_beta(data))
    res = pd.concat(res, axis=1)
    res.columns = ['vol', 'money', 'tr']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    return res
```

前7行，读取数据，这里笔者的股本数据是从2005年1月到2025年12月的，而分钟行情数据是2004年1月到2026年1月的，为了避免获取不到股本数据，增加了第4-5行的保护代码。

第8-13行，计算收益率、成交量、成交额和换手率。

第13-15行，分别计算用换手率、成交量和成交额计算的个股流动性变换率。

第17-19行，分别计算基于成交量、成交额和换手率的beta因子。

```python
def cal_beta(self, data):
    data = data.replace(np.inf, np.nan)
    data = data.replace(-np.inf, np.nan)
    cols = data.columns.tolist()
    data['x'] = data.mean(axis=1)
    beta = []
    for i in range(data.shape[1] - 1):
        beta.append(self.__cal_beta__(data.iloc[:, [i, -1]]))
    beta = pd.Series(beta, index=cols)
    return beta
```

第2-3行，将无穷大替换为nan，这是因为对数计算的时候0会变成无穷大。

第5行，计算市场流动性变化率。

第6-9行，按照标的计算beta因子。

```python
def __cal_beta__(self, data):
    data = data.dropna()
    if len(data) <= 60:
        return np.nan
    y = data.iloc[:, 0].values.reshape(-1, 1)
    x = data.iloc[:, -1].values.reshape(-1, 1)
    x = np.c_[np.ones((len(data), 1)), x]
    beta_full = np.linalg.pinv(x.T @ x) @ x.T @ y
    return beta_full[1, 0]
```

这部分就是计算beta因子了，需要注意的是笔者对每日有效时间小于60分钟的标的都返回nan值。

**因子评价**

神奇的事情出现了，之前计算的非流动性相关的因子，基于成交量、成交额或换手率计算的，相关性差距挺大的。

但是，这一次，它们之间的差距很小，如果计算的是截面斯皮尔曼相关系数的话，这三个因子的相关性都是1。

笔者也对这三个因子都进行了评价，结果发现其因子评价结果用肉眼也无法看出任何差异。

虽然，这三个因子的相关性很高，但是用它们做差并没有出现奇迹。
