本文，笔者复现的因子来自2024年6月5日发布的研报《高频研究系列八—关联信息中的 Alpha》中《关联信息中的局部特征：“交易时间”和持续期特征》这一小节介绍的因子。

本来，是打算介绍《关联信息中的全局特征：非线性因果关系》这一小节当中的因子的，但是这一小节有一个概率，叫做转移熵。笔者发现研报和网上的公式不太一样，网上的公式复杂一些，研报的简单些。既然是复现研报，肯定要以研报的为准，但是之前的代码是基于网上的公式写的。所以，就不太对了。

没办法，就先调整了一下，《关联信息中的全局特征：非线性因果关系》这一小节的因子就下一篇文章介绍了。

本文介绍的因子叫做交易时间下的基础性因子。

该方法通过成交量，将240分钟划分为50个交易时间。这个思想和之前的分域非常像。

第一步，计算“交易时间”阈值。即，当日总成交量除以50。

第二步，计算“交易时间”价格。从第一分钟开始，我们逐步累加成交量，若成交量累加值达到“交易时间间隔”时，说明当前“交易时间”达到“一个时刻”。此时，我们将囊括在内的所有分钟收盘价按照分钟成交量占比加和，得到当前“交易时间”的收盘价。这里需要注意的是，若在 t 自然分钟时，加上分钟成交量后将超过阈 ，我们仅 算达到阈 部分的成交量，剩余部分为下一个交易分钟的成交量。

有了成交量就能计算收益率了。根据研报的说法，通过这样的操作会将收益率变得更接近正态分布。

有了收益率就有了两个因子，收益率的均值cutVol\_rtnMean，收益率的方差cutVol\_rtnVar。

此外，每个区间的持续时间也能得到两个因子。持续时间的均值cutVol\_timeMean，持续时间的方差cutVol\_timeVar。这两个因子中的后一个，能反映标的每日成交量的集中程度，如果成交量集中于某些时间，那么方差会很大。至于前一个，笔者认为它在所有的标的上面没有实质性差异，因为大家都是240分钟划分为50个交易区间的，这均值理论上应该是一样的。实际计算出来的结果都在4.8左右，因为有些标的可能会出现最后几分钟成交量为0的情况。

所以，这里就出现了理解的问题了。研报中的原文是我们进一步统计“交易时刻”的时间间隔序列的均值和方差，作为“交易时间”下的时间特征均值因子和方差因子。

时间间隔序列应该就是每个区间的持续时间了，但是这样计算均值就没有意义了。笔者反复思考，觉得还可以计算时间间隔的一阶差分，然后再计算均值和方差，这样的话就不会出现均值都相等的情况了。

```python
def cal_factors(data):
    close, time_list = [], [0]
    thd = round(np.nansum(data[:, 0]) / 50, 2)
    if thd == 0:
        return [np.nan] * 6
    cur_vol, cur_close = 0, 0

    for i in range(len(data)):
        diff = round(thd - cur_vol, 2)
        if data[i, 0] is None:
            continue
        if diff - data[i, 0] <= 0.01:
            cur_close += data[i, 1] * diff / thd
            close.append(cur_close)
            cur_vol = (data[i, 0] - diff)
            time_list.append(i + 1)
            while thd - cur_vol <= 0.01:
                time_list.append(i+1)
                close.append(data[i, 1])
                cur_vol -= thd
            else:
                cur_close = cur_vol * data[i, 1] / thd
        else:
            cur_close += data[i, 0] * data[i, 1] / thd
            cur_vol += data[i, 0]
    time_list = np.diff(time_list)
    time_list_2 = np.diff(time_list)
    ret = np.diff(close) / np.array(close)[:-1]
    return [np.mean(time_list), np.var(time_list), np.mean(ret), np.var(ret),
            np.mean(time_list_2), np.var(time_list_2)]
```

第3行，计算交易时间阈值。

第4-5行，是第三次修改的时候加上的，不然会内存爆炸。

第6-18行，就是从第一分钟开始累加，累加到阈值（10-23行）了就将交易时间价格和时间添加到close和time\_list两个列表中，否则（21-23行）继续累加。

第15-20行，是排查时间均值不相等的时候修改的，刚开始没有，第一次修改改成了if，第二次修改改成了while，第四次修改了第12和第17行，将直接大小比较换成了差值小于0.01（这样的话第3行和第9行可以去掉round了，用round的话大概会慢3倍左右）。不得不说，笔者对自己的代码越来越没有信心了，也许之前的代码也有一些错误，希望各位大佬看到之后能指点一下笔者，谢谢大家。

第24行，计算时间间隔。

第25行，计算时间间隔的一阶差分。

第26行，计算收益率。

这段代码的重点在于第3行和第7行，浮点数的比较。之前没有加round，导致漏掉了最后一个区间。这个bug也是通过交易时间的均值不相等看出来的。当然，这里主要的问题是刚好划分为50个区间，如果不加round的话，就只有49个区间了。如果是取大于门限的第一个区间的话（类似于之前介绍的聪明钱因子，是成交量占比达到20%），就不需要round也行。
