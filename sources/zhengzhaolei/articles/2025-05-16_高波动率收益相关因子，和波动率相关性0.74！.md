本来，这篇文章应该是要介绍残差的一阶自相关性的。

但是，残差的计算需要用到线性回归，然后相对来说比较慢。

昨天睡觉前开始运行代码，结果5点20的时候电脑死机了，一点结果也没保存。于是，就换了一个因子。

这个因子还是来自《高频系列研究六——时序信息中的Alpha》。

这个因子来自研报中的第四小节，《条件分布基础因子》。

这个因子先要计算过去30分钟滚动5分钟收益率的波动，然后筛选出波动率超过80%分位数的时刻，最后计算这些时刻收益率的均值，作为当日的因子值。

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close'])
    date_str = file_name.split('.')[0]

    ret = data.to_dataframe('close').pct_change(5)
    sigma = ret.rolling(30).std()
    thd = sigma.quantile(0.8)
    flag = sigma > thd
    ret = (ret * np.where(flag, 1, np.nan)).mean()
    ret.name = pd.to_datetime(date_str) + timedelta(hours=15)
    return ret
```

这个因子的计算就比之前简单很多了。

第8行，计算滚动5分钟收益率。

第9行，计算过去30分钟滚动5分钟收益率的波动率。

第10行，计算波动率80%分位数的门限。

第11-12行，计算波动率超过80%分位数时刻的5分钟收益率的均值。

在因子评价的时候，笔者发现，用均值合成的因子IC绝对值低于用标准差合成的因子。但是，考虑到用标准差会和21日波动率的相关性高，因此先进行一个相关性的测试，看看这个高波动收益率波动因子和传统的21日波动率因子的相关性到底会有多高呢？

总结：这个因子有点难评阿，用均值合成的话IC绝对值不高，分层回测也不好。用标准差合成的话，也就IC绝对值高点，分层回测可以说是一般，同时这个因子和传统的21日波动率有着极高的相关性。由于该因子用到了分钟级别的数据，其计算起来的复杂度也是远高于传统的21日波动率的。
