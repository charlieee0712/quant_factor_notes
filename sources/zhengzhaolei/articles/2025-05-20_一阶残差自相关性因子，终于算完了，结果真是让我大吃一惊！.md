这个因子的核心就是如何计算残差。

这里，笔者参考了之前复现Barra CNE6中因子的时候代码。

```python
@staticmethod
def lr(data):
    x = data.iloc[:-1]
    y = data.iloc[1:]
    Y_full = y.dropna(axis=1)
    idx_full, Y_full = Y_full.columns, Y_full.values
    X_full = np.c_[np.ones((len(x), 1)), x.loc[:, idx_full].values]
    beta_full = np.linalg.pinv(X_full.T @ X_full) @ X_full.T @ Y_full
    resi = Y_full - X_full @ beta_full
    resi = pd.DataFrame(data=resi, columns=idx_full)

    for c in set(y.columns) - set(idx_full):
        y_lack = y.loc[:, c]
        x_lack = x.loc[:, c]
        idx = x_lack.dropna().index.intersection(y_lack.dropna().index)

        if len(idx) < 30:
            resi[c] = np.nan
            continue
        X_lack = x_lack.reindex(idx)
        Y_lack = y_lack.reindex(idx).values
        X_lack = np.c_[np.ones(len(X_lack)), X_lack.values]
        beta_lack = np.linalg.pinv(X_lack.T @ X_lack) @ X_lack.T @ Y_lack
        resi[c] = y_lack - np.c_[np.ones(len(x_lack)), x_lack.values] @ beta_lack
    return resi
```

对于没有缺失值的情况，直接一次性批量计算完成，对应的代码是3-10行。

对于存在缺失的情况，不同的标的缺失值出现的时间点大概率是不一致的，因此需要单独计算每一个标的的，对应的代码12-24行。

这里需要注意的是，x\_lack和X\_lack，小写的是原始数据包含了nan值，大写的去掉了Nan值。最后在计算残差的时候用的是小写的，因为用大写的话还要把缺失值补充进来，不然没办法给resi这个dataframe按照列赋值了。

从研报中给出的因子评价来看，一阶残差的自相关性确实会比一阶自相关性差，但是差距也不是挺大。可是，笔者在复现的时候发现，这一系列因子可以说是完全不能用。

无论是收益率计算的一阶残差自相关性rtn\_rho，还是成交量占比计算的一阶残差自相关性vol\_rho，甚至是成交量占比计算的一阶残差自相关性vol\_rho与成交额占比计算的一阶残差自相关性money\_rho的差值，都很差。

以diff这个因子为例，这个因子用标准差合成的因子好于用均值合成的。

但是，即使这个看起来好了那么一点点的因子，实际上也可以说是非常差的。

虽然，这个因子每一年的IC都是负数，但是IC绝对值最大的时候，也没有超过0.01啊！
