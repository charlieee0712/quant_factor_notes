本文，笔者复现的研报是2025年7月20日，由开源证券的魏建榕和王志豪两位老师发布的《市场微观结构研究系列（27）：高频成交量的峰、岭、谷信息》。

在这篇研报中，一共提出了11类，多达20个有效的因子。因此，在复现的过程中，笔者需要分几篇文章来慢慢介绍了。

这些因子的计算都是在一个总的构思下的，那就是将成交量划分成峰、岭和谷这三个不同的部分。

在划分峰岭谷之前，先要计算喷发成交量。所谓喷发成交量就是成交量大于均值+1倍标准差的时刻，不过这里的均值和标准差不是以同一日的数据来计算的，而是以过去21个交易日同分钟的数据来计算的。

根据喷发成交量就能将成交量划分成峰、岭、谷了。

横看成岭侧成峰，地理 中将孤耸的高山称为“峰”，将连绵的高山称为“岭”，将高山之间的低洼地带称为 “谷”。因此，本报告中，我们将温和成交量称为“量谷”，将连续喷发成交量称为 “量岭”，将孤立喷发成交量称为“量峰”。

通过上述步骤，就完成了成交量的划分。

划分完成之后，研报中一共提到了11个大类因子，分别是分钟数、分钟收益、相对加权价、加权价格分位点、时间间隔统计特征、加权价格比值、成交额比值、成交额跟随比例、成交额敏感度、成交额corr和同时点统计数corr。

本文，笔者将介绍第一个因子，也就是分钟数。其计算代码如下：

```python
def process_single_day(self, idx):
    if idx < 20:
        return pd.DataFrame({})
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    vol = []
    for i in range(idx-20,idx+1):
        file_name = self.files[i]
        full_path = os.path.join(self.file_pth, file_name)
        data = BaseDataLoader.load_data(full_path, fields=['volume'])
        vol.append(data.to_dataframe('volume'))
    flag = self.cal_flag(vol)
    peak = (flag == 1).sum()
    ridge = (flag == 2).sum()
    valley = (flag == 0).sum()
    res = [peak, ridge, valley]
    res = pd.concat(res, axis=1)
    res.columns = ['peak', 'ridge', 'valley']
    # res['diff'] = res['amt_foc'] - res['vol_foc']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()

def cal_flag(self, data):
    data = pd.concat(data)
    data['minute'] = data.index.hour * 60 + data.index.minute
    data = data.groupby('minute', as_index=False, group_keys=False).apply(self._cal_flag_)
    flag = data.shift(fill_value=False) | data.shift(-1, fill_value=False)
    flag = np.where(data, np.where(flag, 2, 1), 0)
    return pd.DataFrame(flag, index=data.index, columns=data.columns)

@staticmethod
def _cal_flag_(group):
    group = group.drop(columns=['minute'])
    thd = group.mean() + group.std()
    group = group >= thd
    return group.iloc[[-1]]
```

这段代码一共有三个方法，后面两个cal\_flag和\_cal\_flag\_方法是用来获取喷发成交额对应时刻的flag的。

第27-28行，对不同交易日同一分钟的数据进行groupby操作。然后，通过\_cal\_flag\_方法计算当前时刻的成交量是不是喷发了，返回一个bool型的dataframe（0代表没有喷发，1代表喷发），该dataframe的长度为1。

第29行，获取连续的喷发时刻，将其置为True，其余置为False。

第30行，data中的False都置为0，代表谷；True就要再分情况了，如果flag对应位置也为True则置为2，代表岭‘其余为0，代表峰。

得到这样一个由0、1和2构成的dataframe之后，后续的操作就简单了。

对于本研报复现的因子来说，那就是统计一下0、1和2这三个不同的数值的个数就好了。

总结：研报当中表示峰和岭这两个因子是有效的，而谷是无效的。从笔者的复现结果来看，似乎也是这样的。

峰的表现最好，IC绝对值最高（2020年和2023年这两年的IC绝对值超过了0.08），分层回测的单调性也是最好。虽然，净值曲线存在相交的情况（这个情况可以通过等权分层回测或者拉长回测周期从2013年开始回测来改善），但至少最后的净值排序和因子值大小的排序相关性为1。

岭的表现次之。首先是IC上，岭只有一年的IC绝对值超过了0.07；然后是分层回测方面，最终净值最高的是因子值第二小的一组。

最后是谷，IC绝对值最大的一年也没有超过0.05，而最小的一年都没能超过0.01。至于分层回测，那也是差得一塌糊涂啦！因子值最大的一组和因子值最小的一组收益率是最低的。

值得一提的是，上述因子在合成月度因子的时候都是用的过去21个交易日的均值来合成的。如果用标准差的话，谷的IC表现会略好一点，有两年的绝对值超过了0.06，但是分层回测就从开始相交到了最后。岭的表现稍微下降，IC绝对值最高的一年也没有超过0.07。至于峰的话，那就是大幅度的下降了，IC最高的一年也没能超过0.03。
