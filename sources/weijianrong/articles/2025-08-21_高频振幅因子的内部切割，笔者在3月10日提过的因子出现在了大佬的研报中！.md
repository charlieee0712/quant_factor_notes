不得不说，开源证券的魏建榕老师真的是太努力了，2025年7月20日发布了一篇研报，里面有10多个因子，笔者直到现在也没复现完。

结果不到20个交易日的时间，人家又发布了一篇研报《市场微观结构系列（30）：高频振幅因子的内部》。

这个因子是在理想振幅因子上做的改进，在2020年的时候他们发布的研报《振幅因子的隐藏结构》中第一次提到了这个因子。![](images/2025-08-21_高频振幅因子的内部切割，笔者在3月10日提过的因子出现在了大佬的研报中！/001.jpg)

当时的这个因子是日频的，笔者在一年多以前复现过，那时候的文章写得很烂，至少比现在要烂。[理想振幅因子](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247484501&idx=3&sn=98c33e11b687518f94bb0c6e850c2af3&scene=21#wechat_redirect)

后来，一位读者问笔者要这个因子的代码，笔者找不到了，于是又复现了一遍，用分钟级别的数据来复现了一下。[理想振幅因子，看起来还行，至少比欧氏距离计算的惊恐度强](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247485670&idx=1&sn=b3ddff1fbb789155d53f431c129eb1f3&scene=21#wechat_redirect)

没想到，在笔者阴差阳错的用分钟级数据实现了理想振幅因子之后，这个因子居然出现在了研报当中。

不过，研报中的这个因子，和笔者当时复现的还是有区别的。

研报中分成了两个，第一个是分钟理想振幅因子的构造。这个就是用过去20个交易日的分钟数据放到一起来构造理想振幅因子。

第二个则是，日内振幅切割因子，这个因子的构造是用每日的分钟数据构造一个每日的振幅因子，然后用过去10个交易日每日的振幅因子的均值或者标准差来构造新的因子。这个思想和笔者上面提到的第二篇文章是类似的。

不过，研报中提到了多种方式将振幅划分成high和low，包括收盘价、振幅、成交额、1分钟收益和5分钟收益。从研报的结果来看，成交额划分的在IC上的表现是最好的。因此，在这里，笔者主要复现成交额划分的，然后顺便再复现一下收盘价划分的，两者进行一个比较！

接下来，详细介绍一下这个因子的计算步骤，并结合代码具体说明。

第一步，计算每分钟的振幅，即最高价/最低价-1。

第二步，根据某一指标，将数据划分为两个区间，指标较高的q%区间内的振幅均值作为high，指标较低的q%区间内的振幅均值作为low。这里的9可以是5~50中的任何一个数，在分钟理想振幅因子中，作者提到25%比较好。

high和low相减，就得到了日理想振幅因子。后续的均值和标准差的计算就不提啦。

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'close', 'high', 'low', 'turnover'])
    date_str = file_name.split('.')[0]

    vol = data.to_dataframe('volume')
    amp = (data.to_dataframe('high') / data.to_dataframe('low')) - 1
    money = data.to_dataframe('turnover')
    close = data.to_dataframe('close')

    res = []
    for s in [vol, money, close]:
        thd = s.quantile([0.3, 0.7])
        amp_series = amp.where(s >= thd.iloc[1]).mean(axis=0) - amp.where(s <= thd.iloc[0]).mean(axis=0)
        res.append(amp_series)
    res = pd.concat(res, axis=1)
    res.columns = ['vol', 'turnover', 'close']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

核心的计算代码就是第16行。
