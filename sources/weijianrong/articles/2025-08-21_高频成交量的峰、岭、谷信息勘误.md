之前，笔者复现了开源证券的魏建榕和王志豪两位老师发布的《市场微观结构研究系列（27）：高频成交量的峰、岭、谷信息》。

这篇研报中提到的因子有很多，直到现在笔者写了四篇文章了，依旧没有复现完。

今天，笔者在准备继续复现的时候，发现了一个代码中的错误。这个错误将峰、岭、谷三者的顺序弄反了。具体是这样的。

```python
flag = data.shift(fill_value=False) | data.shift(-1, fill_value=False)
flag = np.where(data, np.where(flag, 2, 1), 0)
```

先看这两行代码，这两行代码将成交量不激增的时刻置为0，连续激增时刻置为2，非连续激增时刻置为1。

但是，笔者在输出dataframe的时候犯错了，习惯性的将第一个认为是峰、第二个认为是岭、第三个认为是谷了。错误的是下面这段代码的第四行。

```python
for i in range(3):
    res.append((ret * np.where(flag == i, 1, np.nan)).sum())
res = pd.concat(res, axis=1)
res.columns = ['peak', 'ridge', 'valley']
```

![图片](images/2025-08-21_高频成交量的峰、岭、谷信息勘误/001.png)

那么，0对应谷，1对应峰，2对应岭。

但是，笔者在计算的时候变成了0对应峰，1对应岭，2对应谷了。

也就是说，笔者因子评价中的量峰其实是量谷，量岭其实是量峰，量谷才是量岭。

基于此，之前的结论应该是这样的。

第一篇文章中，笔者说峰和岭这两个因子是有效的，而谷是无效的，结论和研报一致。（这篇文中从文章中贴出来的代码来看是对的）。

从第二篇文章开始，文章中贴出来的代码就出现了问题。

第二篇文章中，实际有效的因子应该是量岭。（这么看来，只有第二篇文章的结论和研报有出入，于是笔者决定重新跑一遍。遗憾的是，重跑一遍之后，）

第三篇文章中，实际上从IC的表现来看是量谷>量岭>量峰，可以说是和研报的结论是一致的。

第四篇文章中，实际上表现不错的量峰和量谷因子，这也可以说是和研报结论一致的。

于是，就有了这篇勘误的文章。笔者在复现的时候太不严谨了，还望各位大佬多多担待。
