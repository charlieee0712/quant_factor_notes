在上一篇文章中，笔者介绍了量峰分钟数、量岭分钟数和量谷分钟数这三个因子。

其中，量谷分钟数的表现是最差的，量岭分钟数的表现稍微好了一点，最好的则是量峰分钟数。

本文，笔者将继续研究这篇研报中提到的第二个大类的因子，分钟收益率。

这个因子的计算很简单，就是在上一篇研报中提到的峰、岭和谷的划分的基础上，计算过去21个交易日的累计收益率作为量峰分钟收益因子、量岭分钟收益因子和量谷分钟收益因子。

```python
def process_single_day(self, idx):
    if idx < 20:
        return pd.DataFrame({})
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    vol = []
    for i in range(idx-20,idx+1):
        file_name = self.files[i]
        full_path = os.path.join(self.file_pth, file_name)
        data = BaseDataLoader.load_data(full_path, fields=['volume', 'close', 'open'])
        vol.append(data.to_dataframe('volume'))
    flag = self.cal_flag(vol)
    ret = np.log(data.to_dataframe('close') / data.to_dataframe('open'))
    ret = ret.reindex(flag.columns, axis=1)
    res = []
    for i in range(3):
        res.append((ret * np.where(flag == i, 1, np.nan)).sum())
    res = pd.concat(res, axis=1)
    res.columns = ['peak', 'ridge', 'valley']
    # res['diff'] = res['amt_foc'] - res['vol_foc']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

这段代码中，13行之前的部分是读取数据和分类（将每分钟分为峰、岭和谷）。这些在上一篇文章中都有过介绍了，就不赘述了。

真正不一样的代码从第14行开始。

第14行，计算对数收益率，这里每分钟的收益率是用收盘价/开盘价来计算的。取对数再求和，就相当于累积收益率。

第15行，让收益率dataframe的列和flag的列保持一致，方便后续计算。

第16-20行，计算量峰分钟收益、量岭分钟收益和量谷分钟收益这三个因子，并拼接为一个dataframe。

总结：研报中提到这三个因子只有量谷分钟收益是有效的，其他两个因子都是无效的。这一结论和笔者的复现结果基本一致。

从IC分析上来看，量谷分钟收益的IC绝对值是最高的，量岭分钟收益率次之，量谷分钟收益率最差。

但是从分层回测上来看，又有一点点区别了。表现最差的是量峰分钟收益率，这一点是没有任何疑问的。但是，量谷分钟收益和量岭分钟收益的分层回测结果比较起来，似乎量岭分钟收益的表现更好（单调性更好，相交的情况也更少）。

上述因子都是过去21个交易日累加，也就是和取21个交易日的均值类似。

因此，还可以通过过去21个交易日的标准差来合成。

和上述结果相比，只有量谷分钟收益用标准差合成之后的效果有所提升，其余的都和之前的结果没有什么区别，甚至更差。所以，此处仅展示一下量谷分钟收益用标准差合成的结果。
