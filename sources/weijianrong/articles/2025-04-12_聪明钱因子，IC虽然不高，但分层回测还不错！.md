本文介绍的因子，参考了开源证券魏建榕老师在2020年发布的研报《聪明钱因子模型的2.0版本》。

为什么说是参考呢，因为笔者在实现的时候偷懒了，和研报的内容并不是完全一样的。当然这都是借口，真实的情况是代码写错了。

研报种因子的计算分为四步。

第一步，对于选定股票，回溯取其过去10个交易日的分钟行情数据，构造指标S。该指标是（t分钟收益率分绝对值）/（t分钟成交量的0.25次方）。

第二步，将分钟级数据按照S指标从大到小排序，取成交量累积占比20%的分钟，视为聪明钱交易。

第三步，计算聪明钱交易的成交量加权平均价，即vwap价格；同时计算所有交易的成交量加权平均价。

第四步，计算聪明钱的vwap/所有交易的vwap。

对于这个因子，具体的说明如下：

聪明钱因子是通过在分钟行情数据的价量信息中识别机构参与交易的多寡而构建的跟踪聪明钱交易的选股因子，而聪明钱的交易是基于“单笔订单数量更大、订单报价更为激进”的交易特征来识别的。

笔者在实现的时候，没有考虑过去10个交易日的行情数据，而是每个交易日单独计算了一个聪明钱因子，具体实现如下。

```python
def cal_factors(data):
    data.columns = ['s_t', 'volume', 'turnover']
    data.sort_values(by='s_t', ascending=False, inplace=True)
    total_vol = data['volume'].sum()
    total_money = data['turnover'].sum()

    thd = total_vol * 0.2
    smart_vol = 0
    smart_money = 0
    for i in range(len(data)):
        smart_vol += data.iloc[i, 1]
        smart_money += data.iloc[i, 2]
        if smart_vol > thd:
            break
    vwap_smart = smart_money / smart_vol
    vwap = total_money / total_vol
    return [vwap_smart, vwap_smart / vwap]
```

第3行，按照指标S排序。

第4-5行，计算标的单日的总成交量和总成交额。

第7行，将总成交量的20%作为门限。

第8-14行，计算聪明钱的成交量和成交额。

总结：本文因子评价展示的因子是过去21个交易日聪明钱因子的均值。虽然，用21天的标准差比用均值的IC绝对值更高，但是用均值的话分层回测的曲线看起来好一些，至少没有那么多相交的情况。此外，该因子笔者的计算方式和研报不同，我们有理由相信，如果按照研报的方式计算，那么分层回测的单调性会更好。这一猜想，笔者会在明天的文章中证实一下。
