**尾部风险**

在尾部风险这个专题中，笔者写了一系列文章来介绍与之相关的因子。

比如：最近的[尾部Beta，原来计算Beta不一定要用最小二乘！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247496452&idx=1&sn=b82e65478cde49cda26cb543c9d5dd9b&scene=21#wechat_redirect)，还有很久很久以前的[CVaR因子，IC表现不错，分层回测有待提升！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247489552&idx=1&sn=d44ca3f3be45b9c25d78ee4c022d980a&scene=21#wechat_redirect)。

本以为，这个专题已经介绍的差不多了。

没想到，在2022年的《因子日历》上，笔者又找到了一个和尾部风险相关的因子。

这个因子是老外在2014年的论文中介绍的。

通常来说，笔者不太愿意复现老外的因子，因为它们在大A的表现可以用拉完了来形容。比如：[怕自己邪修走火入魔，找了老外论文里的因子复现，结果让人大吃一惊！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247491576&idx=1&sn=766f985afd31848889fd6e8450e12a26&scene=21#wechat_redirect)

可是，在2022年《因子日历》上出现的老外的因子似乎有所不同，比如[特异市值，加上“特异”两个字之后，居然变成了一个宝藏因子？](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247496525&idx=1&sn=6673ecf499c5dbc80e6f74e48799fccb&scene=21#wechat_redirect)

在这种情况下，笔者决定再相信老外一次。

**计算步骤和代码**

这个因子的计算用到了时序回归，然后取的是beta值，听起来有那么一点儿像尾部Beta这个因子。

但是，从笔者的感受来看，这两个因子的差异还是很大的。

**1**

**计算步骤**

以收益率为被解释变量，截面的CVaR值/VaR值为解释变量，进行时序回归，然后取Beta值作为因子。

VaR值：截面收益率25%分位数的值。

CVaR值：截面收益率小于25%分位数的均值。

**代码**

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    cur_time = pd.to_datetime(date_str) + timedelta(hours=15)
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close']).to_dataframes()
    rtn = data['close'].pct_change()
    q = rtn.quantile(0.25, axis=1)
    flag = rtn < q.values.reshape(-1, 1)
    rtn['mkt'] = (rtn * np.where(flag.values, 1.0, np.nan)).mean(axis=1) / q
    rtn = rtn.replace(np.inf, np.nan)
    rtn = rtn.replace(-np.inf, np.nan)
    tail_beta = []
    for i in range(len(rtn.columns) - 1):
        tail_beta.append(self.cal_resi(rtn.iloc[:, [i, -1]]))
    tail_beta = pd.Series(tail_beta, index=rtn.columns.tolist()[:-1], name=cur_time)
    return tail_beta
```

前6行，读取数据。

第7行，计算收益率。

第8行，计算截面VaR值。

第9-10行，计算截面CVaR值/VaR值。

第11-12行，将inf置为nan。

第13-17行，计算因子值。

```python
@staticmethod
def cal_resi(data):
    data = data.dropna()
    if len(data) < 2:
        return np.nan
    lr = LinearRegression()
    lr.fit(data.iloc[:, 1:], data.iloc[:, 0:1])
    return lr.coef_[0][0]
```

但是，笔者短暂的量化生涯主要是和券商打交道的，他们非常厌恶小票暴露，所以笔者几乎每一个因子评价都进行了行业市值中性化（[特异市值，加上“特异”两个字之后，居然变成了一个宝藏因子？](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247496525&idx=1&sn=6673ecf499c5dbc80e6f74e48799fccb&scene=21#wechat_redirect)，这篇文章没做是因为在计算因子的时候已经有行业市值中性化的思想了）。或许，在一次次和券商打交道的过程中，不知不觉塑造了自己的量化观吧。

**总结**

这个因子在不进行行业和市值中性化的情况下，其分层回测的结果看起来是不错的。

同时，笔者也大胆猜测，如果在分层回测的时候采用等权，而不是市值加权也会得到不错的分层回测结果。

但是，它不符合笔者的量化观。
