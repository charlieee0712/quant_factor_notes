**单一成交额占比熵**

本文，笔者要复现的因子来自长江证券的首席分析师覃川桃老师，她在2020年发布的研报《高频因子（八）：高位成交因子——从量价匹配说起》中提到了一个叫做单一成交额占比熵的因子。

关于这个因子的逻辑，2024年的《因子日历》中是这样描述的。

> 以单一成交额占比作为每一个时间段状态的发生概率，以熵的定义刻画成交的混乱程度，即得到单一成交额占比熵因子；当因子值越小时，体系越混乱，成交量占比和收盘价占比排序相对一致（单位成交额占比彼此之间存在较大分歧），成交在价格高位较为密集，易造成股价的高估。

**计算步骤和代码**

这个因子的虽然名字当中出现了成交额三个字，但是它在计算的时候并没有直接使用成交额，而是使用了t分钟的收盘价和成交量。

**1**

**计算步骤**

第一步，计算t分钟收盘价与全天收盘价之和的比值。

第二步，计算t分钟成交量与全天成交量的比值。

第三步，将上述两个比值相乘然后计算熵。

**2**

**代码**

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'close']).to_dataframes()
    money = data['close'] * data['volume']
    money = money / (data['close'].sum() * data['volume'].sum()).values.reshape(1, -1)
    res = -(money * np.log(money)).sum()
    res.name = pd.to_datetime(date_str) + timedelta(hours=15)
    return res
```

这个因子的代码可以说是非常简单的了，感觉没有什么需要特别说明的地方了。

**因子评价**

![](images/2026-01-24_单一成交额占比熵，这个因子用30分钟数据计算，效果炸裂！/002.png)

在《因子日历》中，这个因子是用5分钟数据计算的，笔者试了用1分钟的数据计算，效果很不好。

于是，就顺便把15分钟、30分钟和60分钟频率的数据也用了起来，发现用30分钟和60分钟数据计算的因子IC表现最好（相关性也证实了这一点，这两个因子的相关性最高，接近1），而用30分钟数据计算的因子分层回测也是非常的不错。

所以，下面展示的因子评价结果是基于30分钟数据计算得到的因子。

同时，这个因子用均值合成的在IC和分层回测上的表现都好于用标准差合成的。
