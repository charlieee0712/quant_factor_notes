这段时间，笔者复现了几个因子。

一个是，根据拐点来计算日内反转，这个因子是期货里面的，然后用到股票上的效果并不是很好。

另一个是，APM因子，这个因子的计算有点复杂，耗时也有点长。第一次计算出来发现效果不好，因为用错数据了，然后修改了之后再次计算，只能说比第一次的效果好了一点点，但还是不太行。

于是，接着复现了第三个因子，一致交易因子。

这个因子的效果说不上太好，但是在最近复现的这三个因子当中，它属于效果最好的了，所以就拿出来和各位大佬一起讨论讨论。

![Image](images/2025-06-17_一致交易因子，参数对结果的影响有点大/001.png)

这个因子来自光大证券刘均伟老师在2018年1月3日发布的研报《多因子系列报告之九——一致交易因子：挖掘集体行为背后的收益》。

首先，我们来看看这个因子背后的逻辑。

集体一致交易行为预示交易机会，如果一支股票在一段时间内大部分的交易属于集体一致交易行为，趋势性显著，则很可能这支股票在这段时间内有新信息进入，正在经历市场消化过程，未来股价很可能产生大幅变化，为交易者创造交易机会。

然后，看看如何构造这个因子。

首先，需要判断一根k线是否是实体K线。完美的实体K线应该是没有上下影线的，这种K线就要求abs(收盘价-开盘价）=（最高价-最低价）。显然，这样的条件太苛刻了，很难达成。

于是，研报中提到只需要K线满足abs(收盘价-开盘价）<=alpha\*（最高价-最低价）即认为这根K线是实体K线。

然后，计算实体K线的成交量之和与标的当日总成交量的比值便得到了一致交易因子。

这里，需要注意的是alpha的设置会影响到因子的计算。所以，笔者在复现的时候对alpha从0.1到1，每间隔0.1进行了一次遍历。当alpha等于1的时候，如果按照第二种方式计算实体K线的话，那么所有的K线都是实体K线，计算出来的因子将没有任何意义。因此，当alpha等于1的时候，将退化为第一种苛刻的计算方式。具体代码如下：

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close', 'open', 'high', 'low', 'volume']).to_dataframes()
    close = np.abs(data['close'] - data['open'])
    high = data['high'] - data['low']
    total_vol = data['volume'].sum()
    res = []
    cols = []
    for thd in range(10, 110, 10):
        if thd < 100:
            flag = close <= thd * high / 100
        else:
            flag = close == high
        tmp_vol = data['volume'] * np.where(flag, 1, np.nan)
        res.append(tmp_vol.sum() / total_vol)
        cols.append('tcv_{}'.format(thd))
    res = pd.concat(res, axis=1)
    res.columns = cols
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

第6行，计算abs(收盘价-开盘价）。

第7行，计算最高价-最低价。

第8行，计算总成交量。

第11-18行，遍历不同的alpha计算一致交易因子。其中，第13-15行，根据alpha的大小选择计算方式，alpha不等于1时，按照不等式计算；alpha等于1时，按照最严苛的方式来计算。
