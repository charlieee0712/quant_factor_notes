本文不复现研报，也不复现论文，更不用大模型来生成因子。

而是，要自己瞎编一个因子。

当然，说是瞎编，可能在研报中或者其他一些大佬的文章中出现过了。但是，笔者没见到过，就当是自己瞎编的好了。

这个因子参考了量峰谷岭信息的研报，判断一下当前分钟的成交量（成交额）是否高于过去21个交易日同分钟的成交量（成交额）的均值加1倍标准差。

如果，满足这一条件，则置为1，否则为0。

然后计算各自所占的分钟数以及两者之间的相关性，得到三个因子。

具体的计算代码如下：

```python
def process_single_day(self, idx):
    if idx < 20:
        return pd.DataFrame({})
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    amt = []
    vol = []
    for i in range(idx-20,idx+1):
        file_name = self.files[i]
        full_path = os.path.join(self.file_pth, file_name)
        data = BaseDataLoader.load_data(full_path, fields=['turnover', 'volume'])
        amt.append(data.to_dataframe('turnover'))
        vol.append(data.to_dataframe('volume'))
    amt = self.z_score(amt)
    vol = self.z_score(vol)

    res = [amt.sum() / 240, vol.sum() / 240, amt.corrwith(vol)]
    res = pd.concat(res, axis=1)
    res.columns = ['amt', 'vol', 'corr']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
    
 def z_score(self, data):
    data = pd.concat(data)
    data['minute'] = data.index.hour * 60 + data.index.minute
    data = data.groupby('minute', as_index=False, group_keys=False).apply(self._z_score_)
    return data

@staticmethod
def _z_score_(group):
    thd = group.mean(axis=0) + group.std(axis=0)
    group = group > thd
    group = group.iloc[[-1]]
    return group.drop(columns=['minute'])
```

代码其实挺简单的，具体的介绍在之前复现量价关系的高频乐章一文中有所提及。因此，这里就简单介绍一下改动。

最大的改动是在第32行，\_z\_socre\_方法中。之前是实现z\_score标准化的，现在改成了判断值是否大于均值+1倍标准差。

剩下的都是类似的，就不赘述了。

量价相关性

这个因子，笔者给自己想了一个知情交易者的角度来解释。

放量分钟数和放价分钟数的相关性低意味着，成交量上去了，但是成交额并没有上去。此时，股价应该处于一个下跌的趋势中。也就是说，此时有资金在以较低的价格买入，他们很有可能是知情交易者，未来的股价会上涨。
