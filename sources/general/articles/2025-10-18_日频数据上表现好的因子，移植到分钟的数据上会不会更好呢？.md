本文，笔者将一个在日频数据上表现还不错的因子，移植到分钟频率的数据上，看看是不是会进一步提升。

这里，笔者选择的因子就是用行业和市值数据改造之后的时间网络相对中心度。

**因子计算步骤和代码**

**1**

**计算步骤**

关于这个因子的计算步骤，大部分和上一篇文章中是相同的。这里，只能说上一篇文章了，因为这两篇文章是一起发的，所以贴不了链接。

区别在于，分钟级别的数据计算可以有多种方式得到最终的月度因子。

第一种，每日的zscore收益率的平方取均值，然后对过去21个交易日取均值，再取倒数。

第二种，每日的zscore收益率的平方取均值，然后取倒数。最后，在对过去21个交易日的因子值取均值或者标准差。

**代码**

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    idx = pd.to_datetime(date_str) + timedelta(hours=15)
    if idx < self.start or idx > self.end:
        return pd.DataFrame({})
    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, codes=self.codes, fields=['close', 'open'])
    rtn = data.to_dataframe('close') / data.to_dataframe('open') - 1
    ind_data = self.ind_data.loc[idx]
    cap_data = self.cap.loc[idx]
    res = []
    for i in range(len(rtn)):
        df = pd.concat([rtn.iloc[i], ind_data, cap_data], axis=1)
        df.columns = ['rtn', 'ind', 'cap']
        res.append(df.groupby('ind', as_index=False, group_keys=False).apply(self.__cal_factors__))
    res = pd.concat(res, axis=1).T
    res = np.square(res)
    res = res.mean()
    res.name = idx
    return res
```

代码这一块主要展示一下日频数据和分钟级别的数据计算因子时候的区别。

前4行，主要是获取当前交易日的时间。

第5行，主要是用来判断当前交易日的时间有没有行业数据，如果没有的话就返回一个空的dataframe。因为，行业数据的开始时间晚于行情数据，结束时间早于行情数据。

第8-10行，读取分钟级别的行情数据并计算收益率。

第11-12行，按照当前交易日获取行业和市值数据。

第14-17行，循环每一根K线，将分钟级别的收益率数据和日频的行情市值数据拼接为一个dataframe。然后调用\_\_cal\_factors\_\_方法计算因子。

**因子评价**

这个因子合成月度因子的时候，有多种方式，笔者也都尝试一遍，最终表现最好的因子是用第一种方式合成的。

很遗憾，这个因子在分钟数据上的表现不如在日频数据上的。

同时，由于日频的因子表现更好，所以这个因子用60分钟的K线计算的效果也更好。

但是，这里笔者将不展示效果最好的60分钟K线计算的因子了，而是表现不怎样的1分钟K线计算的因子。

和日频数据计算的因子相比，基于1分钟K线计算的因子的IC可以说是全方位的下降，只有2023年一年的IC超过了0.08。而日频数据计算的因子每一年的IC都超过了0.08，2023年更是超过了0.1。

显然，将一个基于日频数据计算且表现不错的因子强行移植到了分钟级别的数据上，不一定会取得更好的效果。
