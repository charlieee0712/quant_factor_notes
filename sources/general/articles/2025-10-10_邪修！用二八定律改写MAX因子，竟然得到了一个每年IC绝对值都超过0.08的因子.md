这篇文章是在MAX因子的技术上进行了改进。日频的MAX因子是取过去21个交易日收益率最大的五天的均值。

那么，本文，笔者将这个因子用到分钟级别的数据上，然后结合一下伟大的“二八定律”。这个定律大家应该很熟了，在投资市场上来说，就是80%的涨幅是在20%的时间里面完成的。

从上面的描述就可以看出，这个因子的IC肯定是负的。

**计算步骤**

这个因子的计算一点儿也不复杂。

就是取过去21个交易日分钟收益率最大的20%的均值。

代码

```python
def process_single_day(self, idx):
    if idx < 21:
        return pd.DataFrame({})
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    price = []
    for i in range(idx-21,idx+1):
        file_name = self.files[i]
        full_path = os.path.join(self.file_pth, file_name)
        data = BaseDataLoader.load_data(full_path, fields=['close'])
        price.append(data.to_dataframe('close'))
    price = pd.concat(price)
    rtn = price.pct_change()
    rtn = rtn.iloc[48:]
    thd = rtn.quantile(0.8)
    rtn = (rtn * (rtn >= thd)).mean()
    rtn.name = pd.to_datetime(date_str) + timedelta(hours=15)
    return rtn
```

代码也不算复杂，第14行之前就是计算过去21个交易日分钟收益率。

这里需要注意的是，笔者在for循环的时候实际上是取了22天的。因为计算收益率的时候要用到收盘价，取21天的话会少一个数据。

正因为如此，所以有了第14行代码。这里从第48个值之后开始取，因为示例是用了5分钟的数据。如果是1分钟的话，把48换成240；15分钟的话，换成16；30分钟的话换成8；60分钟就是4了。

第15行代码，计算80%的分位数。

第16行代码，计算收益率最大的20%时间的收益率的平均值。

**因子评价**

对于这个因子，笔者在1分钟、5分钟、15分钟、30分钟和60分钟的数据上都进行了尝试，然后发现60分钟上的表现是最好的（分层回测上都差不多，但是用60分钟数据计算的IC绝对值更高，所以选择了它）。

所以，下面的因子评价结果展现的也是基于60分钟的数据计算得到的因子。

同时，因为这个因子在计算的时候已经使用了过去21个交易日的数据了。所以，不需要再计算过去21个交易日的均值或者标准差了。当然，要计算也是可以的，就相当于多次平滑了，只不过这样做了之后对结果并没有什么实质性提升。

**总结**

这次的邪修因子只能说是一般。

虽然，它在IC上的表现很好，在回测区间里每一年的IC绝对值都超过了0.08，但是它在分层回测上的表现就有点一般了。

从上面的因子评价中的收益分析就能明显看出这又是一个空头因子，因子值最大的一组的收益率远低于其他四组。看来，想要找一个好的多头因子还是不容易的。

笔者也有个思路，就是将二八定律倒置一下，认为80%的跌幅是在20%的时间里面完成的。这样的尝试也不是太好，表现最好的一组在回测区间里面总的收益率也接近于0了。

也许，在股市里起作用的不一定是二八定律，也有可能是三七定律，所以一个新的邪修的思路就这样诞生了。
