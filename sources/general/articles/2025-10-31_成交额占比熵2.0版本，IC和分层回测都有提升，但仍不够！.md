**成交额占比熵2.0**

笔者最近这段时间发现了一个“神奇”的事情，随着复现的研报因子越来越多啊，奇奇怪怪的想法也就越来越多了。

几天前，瞎弄弄，搞了个聪明钱因子的3.0版本（也可以说是4.0版本，因为在这个因子之前还有一个自己瞎弄的聪明钱因子，但是这不重要）。

最近，又瞎弄了一个因子出来，它叫做成交额占比熵2.0。

关于这个因子，笔者也是受到了两篇研报的启发，这两篇研报笔者在[同价成交量分布中alpha因子，IC和分层回测都还不错](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247488301&idx=1&sn=e9338199d3b01c6c34cdc5b5b38b87f3&scene=21#wechat_redirect)和[成交额占比熵，这个因子较为一般。但是，稍微做下改动，就能天翻地覆！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247489308&idx=1&sn=64d02b40f9c83f9ecdfbcffbbf98b1d0&scene=21#wechat_redirect)这两篇文章中复现了。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    # 加载当日分钟数据
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close', 'turnover'])

    for i in range(len(data.codes)):
        tmp_data = pd.DataFrame(data=data.data[:, :, i], columns=['close', 'turnover'])
        res.append(self.cal_factors(tmp_data))
    res = pd.DataFrame(data=res, index=data.codes.tolist(), columns=['entropy_3', 'entropy_same_close'])
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

这段代码是加载数据，然后调用cal\_factors来计算因子。为了对两种不同的尝试进行区分，笔者用entropy\_3表示第二种尝试，entropy\_same\_close表示第一种尝试。

这个因子仍旧是用过去21个交易日的均值合成月度因子的表现更好，这一点和成交额占比熵因子是一致的。

这个因子和笔者之前复现过的一些因子以及自己邪修或者说是瞎编的因子相比，可以说是平平无奇，但笔者还是将其写了出来，因为它和它的原始因子相比还是有一些提升的。
