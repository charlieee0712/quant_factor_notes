**滴水石穿**

金工诗人曹春晓老师在本月发布了一篇研报《个股日内成交量周期性节奏刻画与“滴水穿石”因子构建——多因子选股系列研究之二十四》，笔者在看到之后立刻进行了复现，[滴水石穿，金工诗人曹春晓老师最新力作，6年IC超0.1，其中3年IC超0.12！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247492920&idx=1&sn=b5cb8732c94a2198a8ab6dfc307b53c7&scene=21#wechat_redirect)。

复现后，发现这个因子的IC值是非常高的，但是市值加权的分层回测结果还是差了那么一点点。

于是，笔者开始思考如何优化一下这个因子，尝试过将因子代码和原理丢给大模型，但是效果也并不能让我满意。

此时，笔者重新阅读了一下研报，理解了这个因子背后的逻辑。它通过统计周期2-5分钟所对应的频域能量占比，这个占比越大意味着成交量的周期越短，也就是可能有人通过TWAP这种方式参与交易，很有可能是机构或者其他知情交易者。

于是，笔者想到了研究生期间学到的一系列谱估计算法。

比如：参数方法MUSIC，它的缺点是需要给定信源数。这个概念比较难解释，简单来说就是要给定一个具体的数值，它会按照这个数值来进行计算，计算后的频谱分量数等于这个给定的数值。此时，如果给定的信源数过少，会丢失掉部分信号，给定的信源数过多，会出现伪峰引入大量噪声。

因此，出现了一系列的非参数方法，在这当中IAA（迭代自适应）方法是比较出色的，也是本文笔者将采用的谱估计算法。

**计算步骤和代码**

关于IAA算法，不是一两句话能介绍清楚的。

如果各位大佬感兴趣的话，可以阅读论文，知网上搜一下一堆基于IAA发表的论文。如果你想看提出这个算的英文论文的话，可以去它们的参考文献里面找。

笔者通过大模型找到的是这篇《Source localization and sensing: A nonparametric iterative adaptive approach based on weighted least squares》，但是不排除大模型出现了幻觉。

**计算步骤**

计算步骤整体上与滴水石穿因子是一样的，就是将傅里叶变换换成了IAA算法。

**代码**

```python
@staticmethod
def cal_factors(vol):
    q = vol.quantile([0.25, 0.75])
    med = q.median()
    iqr = q.iloc[-1] - q.iloc[0]
    vol = vol.clip(lower=med-3*iqr, upper=med+3*iqr)
    vol = vol - vol.mean()
    vol = vol * np.hanning(len(vol))
    fft_cf = np.fft.rfft(vol)
    p_f = np.square(np.abs(fft_cf))
    power_ratio = np.sum(p_f[48:]) / np.sum(p_f[1:])
    vol = vol.values
    target_freqs = np.linspace(0.0, 0.5, 120)
    M = len(target_freqs)
    n = np.arange(240).reshape(-1, 1)
    # 构建傅里叶字典
    A = np.exp(1j * 2 * np.pi * n @ target_freqs.reshape(1, -1))
    # 初始化功率（周期图）
    P = np.abs(A.conj().T @ vol.reshape(-1, 1)).flatten() ** 2 / 240
    P = np.maximum(P, 1e-10)
    for iter_idx in range(5):
        P_symmetric = np.concatenate([P, P[-2:0:-1]])
        r = ifft(P_symmetric).real[:240] * len(P_symmetric)
        # 更新功率
        for k in range(M):
            a_k = A[:, k]
            numerator = np.abs(np.vdot(a_k, vol)) ** 2
            # 快速计算分母
            if iter_idx == 0:
                # 第一次迭代：近似计算
                denominator = 240 * (np.sum(P) + 1e-6)
            else:
                # 第二次迭代：精确一点
                try:
                    R_inv_a = solve_toeplitz(r, a_k)
                    denominator = np.real(np.vdot(a_k, R_inv_a))
                except:
                    denominator = 240 * (np.sum(P) + 1e-6)
            if denominator > 0:
                P[k] = numerator / denominator
        if iter_idx == 2:
            power_iaa_3 = np.sum(P[48:]) / np.sum(P[1:])
    return power_ratio, power_iaa_3, np.sum(P[48:]) / np.sum(P[1:])
```

这段代码第12行之前，都是和之前一样的。

第13行之后，就是用IAA来进行谱估计了，这段代码也是大模型写的。笔者毕业之后就没有从事过相关的行业了，一些知识早就忘记了。

这里需要注意的是，笔者的最大迭代次数设置的是5，同时保留了迭代次数为3的结果。选择5是因为大模型说5次是收敛的迭代次数下限，上限应该是20次，但是太耗费时间了（其实只计算2000天的话应该12小时差不多了，但是笔者想在2025年发布这篇文章），大家有时间的话可以自行尝试。

保留3次的迭代结果，是因为3次是与傅里叶变换数值差距最大的。（调试的时候看出来的）
