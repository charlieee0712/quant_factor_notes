**支撑阻力空间因子**

本文，笔者将再次介绍一个自己胡诌的因子。

这个因子可能是一次自欺欺人上不了实盘也跑不赢指数的因子构造游戏，但是不尝试一下又怎么知道呢？

这个因子的构造过程可以说是参考了两篇研报中的思想，第一篇是净支撑量因子，这个因子笔者在[这个因子竟然连续6年IC绝对值超过0.1！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247490289&idx=1&sn=c067ee3673422b8ce98805203f631ede&scene=21#wechat_redirect)这篇文章中复现过；第二篇则是同价成交量相关的因子，在[同价成交量分布中alpha因子，IC和分层回测都还不错](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247488301&idx=1&sn=e9338199d3b01c6c34cdc5b5b38b87f3&scene=21#wechat_redirect)这篇文章中有过相关的介绍。

**计算步骤和代码**

笔者用同价成交量来确定支撑位和阻力位，然后计算这两个位置对应价格空间。

**1**

**计算步骤**

第一步，对当日分钟级别的数据，按照收盘价聚合得到同价成交量。

第二步，根据同价成交量来确定支撑位和阻力位。支撑位为收盘价小于当日均值的价格中对应成交量最大的那个，阻力位为收盘价大于当日均值的价格中对应成交量最大的那个。需要注意的是，计算收盘价均值的时候，相同的收盘价仅计算一次。

第三步，阻力位价格/支撑位价格-1。

说实话，笔者写这个计算步骤的时候感觉非常痛苦，因为实在不知道该怎么表达。每当这个时候就痛恨自己上学的时候为什么不把语文学好。

**2**

**代码**

```python
@staticmethod
def cal_factors(data):
    data = data.groupby('close').sum()
    if len(data) < 4:
        return np.nan
    data.reset_index(inplace=True)
    flag = data['close'] > data['close'].mean()
    vol_up = (data['volume'] * np.where(flag, 1, np.nan)).idxmax()
    vol_down = (data['volume'] * np.where(flag, np.nan, 1)).idxmax()
    return data.loc[vol_down]['close'] / data.iloc[vol_up]['close'] - 1
```

输入的data是一个dataframe，有两列，一列是收盘价，另一列则是对应的成交量。

第3行，按照成交量聚合，并求和。得到一个index为收盘价，列为volume的dataframe。

第4-5行，对于数据长度少于4的标的都返回nan。

第6-7行，获取收盘价大于均值的flag。

第8行，获取支撑位的索引。

第9行，获取阻力位的索引。

在计算完这个因子的时候，笔者觉得它可能和收益率的绝对值因子相关性比较高。所以，进行了一个相关性的分析。

运气不错，这个因子和日内收益率(rtn\_intraday)的绝对值、日间收益率(rtn)的绝对值的相关性都小于0.5，不算高。

这里之所以计算日内收益率和日间收益率的绝对值是因为我们这个因子都是正的，如果直接和收益率计算相关性的话，那就是接近0了。
