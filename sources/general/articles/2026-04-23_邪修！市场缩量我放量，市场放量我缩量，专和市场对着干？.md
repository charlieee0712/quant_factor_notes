关于这篇文章要聊的因子，是笔者自己瞎琢磨的，用当下热门的词语来形容，那就是邪修。

在研究这个因子的时候，笔者有这样的一个思考。在市场成交量缩量的时候，放量的标的是不是有知情交易者参与，未来收益可能会涨；同时，在市场成交量放量的时候，缩量的标的是不是意味着被忽略了，未来有可能补涨。

基于这个想法，笔者构建了这样的一个因子。

**计算步骤和代码**

这个因子并不是简单地计算标的成交量和市场成交量之间的相关性，所以还是有必要对其计算步骤和代码进行一个简单的介绍的。

**1**

**计算步骤**

第一步，计算市场分钟级别成交量，即全市场所有标的分钟成交量之和。

第二步，对所有标的和市场成交量按照过去21个交易日相同分钟进行zscore标准化。

第三步，计算标的成交量和市场成交量的相关性。

第四步，计算过去21个交易日相关性<0的均值，并除以过去21个交易日相关性的标准差，作为最终的因子。

**2**

**代码**

第13行，调用z\_score方法，计算同分钟zscore标准化后的成交量。

第14行，计算标的成交量和市场成交量的相关性。

```python
def z_score(self, data):
    data = pd.concat(data)
    data['mkt'] = data.sum(axis=1)
    data['minute'] = data.index.hour * 60 + data.index.minute
    data = data.groupby('minute', as_index=False, group_keys=False).apply(self._z_score_)
    return data
```

第2行，将list拼接为dataframe。

第3行，计算市场成交量。

第4行，计算groupby的标签。

第5行，groupby之后，调用\_z\_score\_进行标准化。

```python
@staticmethod
def _z_score_(group):
    group = (group - group.mean(axis=0).values.reshape(1, -1)) / group.std(axis=0).values.reshape(1, -1)
    group = group.iloc[[-1]]
    return group.drop(columns=['minute'])
```

这里，就是一个简单的标准化的代码，没有什么需要特别说明的。

**因子评价**

在开始因子评价之前，笔者想解释一下，为什么用了这么一个复杂的方式来对因子进行低频化的操作。

最开始的时候，笔者发现用标准差低频化的因子IC还不错，但是分层回测展现出来的效果是因子值最大的一组遥遥落后。至于用均值合成的，IC和分层的表现都不好。

就当我准备放弃的时候，想到既然是和市场趋势相反，那就应该统计一下相关性为负出现的概率。于是就有了过去21个交易日相关性<0的均值这样的一个想法。

再考虑到设计这个因子的初衷，很自然的就想到了计算过去21个交易日相关性<0的均值，并除以过去21个交易日相关性的标准差这样一种复杂的构造方式。因为，均值代表相关性<0的概率，标准差代表相关性的稳定性，这两者组合意味着因子值越大，其小于0的概率越高，而相关性也越稳定。简单来说，就是稳定的小于0。

需要注意的是，笔者的回测区间是2018年到2024年，这都是样本内的数据，样本外的测试就留给各位感兴趣的大佬了。
