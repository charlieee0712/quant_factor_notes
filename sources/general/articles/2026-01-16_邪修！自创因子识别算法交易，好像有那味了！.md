**邪修**

![](images/2026-01-16_邪修！自创因子识别算法交易，好像有那味了！/001.jpg)不知道从何时开始，笔者对识别算法交易充满了兴趣。

在去年八月的时候，笔者就有过邪修的经历，[一个瞎编的因子，我想从知情交易者的角度来强行解释一下他](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247489844&idx=1&sn=29b3f77499fa435fb0d8c7ae85b3c5de&scene=21#wechat_redirect)。但是，整体效果并不能让人满意。

在2025年年底的时候，方正证券的曹春晓老师发布了滴水石穿因子，该因子的核心思想也是识别VWAP或者TWAP的算法交易。笔者复现了魔改了，说实话魔改之后的效果确实出乎了我的意料，但还是有缺点。

魔改之前的因子分层不是太好，魔改之后的因子IC下降了一些。换句话说，无论魔改不魔改，笔者都觉得这个因子还不够完美。

因此，有了本文邪修的因子。

**计算步骤和代码**

该因子的构造思想还是想从分钟级别的数据里面寻找VWAP或TWAP交易的痕迹，这类交易一般和大资金或者算法交易有关，研报或者论文中通常将其称为知情交易者。

**1**

**计算步骤**

第一步，取过去21个交易日1分钟量价数据中的成交量、最高价、最低价和收盘价。其中，最高价，最低价和收盘价用来计算分钟级别的振幅。

第二步，对过去21个交易日同分钟的成交量进行zscore标准化，并保留其中zscore大于1的数据，其余置为nan。（这些时刻对应的是放量时刻）

第三步，计算振幅（最高价-最低价）/前一分钟收盘价，然后对振幅进行zscore标准化。这里笔者用了三种不同的方式，分别是过去21个交易日同分钟数据进行zscore标准化，当日数据进行截面标准化以及时序标准化。

第四步，计算标准化后的成交量和标准化后的振幅相关系数，相关性越低代表算法交易参与的概率越大，越来越有可能上涨。

**2**

**代码**

```python
def process_single_day(self, idx):
    if idx < 20:
        return pd.DataFrame({})
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    vol, amp = [], []
    for i in range(idx - 20, idx + 1):
        file_name = self.files[i]
        full_path = os.path.join(self.file_pth, file_name)
        data = BaseDataLoader.load_data(full_path, fields=['volume', 'high', 'low', 'close'])
        vol.append(data.to_dataframe('volume'))
        tmp_amp = (data.to_dataframe('high') - data.to_dataframe('low')) / data.to_dataframe('close').shift()
        amp.append(tmp_amp)
    vol = self.cal_zscore(vol)
    vol = vol * np.where(vol > 1, 1.0, np.nan)
    cur = amp[-1].reindex(columns=vol.columns)
    cur_ts_zscore = (cur - cur.mean().values.reshape(1, -1)) / cur.std().values.reshape(1, -1)
    cur_zscore = (cur - cur.mean(axis=1).values.reshape(-1, 1)) / cur.std(axis=1).values.reshape(-1, 1)
    amp_zscore = self.cal_zscore(amp)
    res = [amp_zscore.corrwith(vol), cur_zscore.corrwith(vol), cur_ts_zscore.corrwith(vol)]
    res = pd.concat(res, axis=1)
    res.columns = ['amp_corr_vol', 'amp_corr_vol_zscore', 'amp_corr_vol_ts_zscore']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    return res
```

第2-13行，获取过去21个交易日的数据，其中第12行，计算振幅。

第14-15行，执行计算步骤中的第二步。

第16行，获取当前交易日的振幅数据。

第17行，对当前交易日的数据进行时序zscore标准化。

第18行，对当前交易日的数据进行截面zscore标准化。

第19行，对过去21个交易日同分钟的振幅数据进行zscore标准化。

第20行，计算三种不用标准化后振幅数据与成交量的相关性。
