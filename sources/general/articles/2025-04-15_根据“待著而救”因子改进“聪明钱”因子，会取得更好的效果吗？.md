周末的时候，笔者介绍了一个“聪明钱”因子。

该因子是通过在分钟行情数据的价量信息中识别机构参与交易的多寡而构建的跟踪聪明钱交易的选股因子，而聪明钱的交易是基于“单笔订单数量更大、订单报价更为激进”的交易特征来识别的。

在更早之前，笔者还介绍了一个叫做“待著而救”的因子，该因子将每日成交量最大的10个时刻作为优势时刻，优势时刻后面的五分钟称为跟随时刻。

那么，将这两篇研报的思想结合一下，就很容易出现一个新的“聪明钱”因子。

将跟随时刻的总成交额/总成交量作为跟随时刻的vwap，然后将跟随时刻的vwap/日频的vwap作为“聪明钱”因子。

在这一改动下，“聪明钱”因子的IC绝对值更高了，但是分层回测的结果也变差了一些。

虽然，这个因子的计算参考了之前两个已经介绍过的因子了，但咱们还是展现一下代码，并做一个简单的说明。

```python
@staticmethod
def cal_factors(data, turnover):
    q = data.nlargest(10)
    q.sort_index(inplace=True)
    top_idx = np.array(q.index.tolist())
    while True:
        diff = np.insert(np.diff(top_idx, 1) > 6, 0, True)
        if np.all(diff):
            break
        top_idx = top_idx[diff]
    top_vol, follow_vol, top_money, follow_money = 0, 0, 0, 0

    for i in top_idx:
        top_vol += q.loc[i]
        top_money += turnover.iloc[i]
        for j in range(1, 6):
            if i + j < 225:
                follow_vol += data.iloc[i + j]
                follow_money += turnover.iloc[i + j]
    return [follow_money / follow_vol, top_money / top_vol, turnover.sum() / data.sum()]
```

该方法输入为两个Series，一个是某标的的当日成交量的数据，另一个则是同一个标的当日成交额的数据。

第3-5行，获取成交量最大的时刻对应的索引。

第6-10行，如果两个优势时刻相差不足五分钟，保留前一个，认为后一个还是前一个的跟随时刻。

后面就是将不同时刻的成交额成交量累加到一起，然后计算不同时刻的vwap。

这里需要注意的是，由于前15分钟的成交量通常是比较大的，所以在处理的时候先将他们去掉了。

跟随时刻“聪明钱”因子

为了和之前的“聪明钱”因子进行对比，这里也是采用了过去21个交易日的均值来合成月频因子的。如果用标准差的话，IC绝对值会更高，从2018年到2024年每年都会超过0.08。不过，分层回测的效果看起来是差不多的。
