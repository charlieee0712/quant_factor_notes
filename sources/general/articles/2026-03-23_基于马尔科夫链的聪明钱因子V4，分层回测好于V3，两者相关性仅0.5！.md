所谓聪明钱因子，就是一段时间的VWAP/整个时间段的VWAP。

在V3中，笔者将其改为一段时间的VWAP/其余时间的VWAP。

因此，魔改这个因子的关键，就是这个一段时间怎么选择。

**计算步骤和代码**

这个因子是基于马尔可夫链的。

通过将分钟收益率大于0划分为状态1，小于等于0划分为状态0来实现的。

其中，先验概率和转移概率是通过当天的市场收益率来估计的。

**1**

**计算步骤**

第一步，将全市场所有标的收益率的均值作为市场收益，并将其划分为0和1两个状态，0对于收益率小于等于0，1对应收益率大于0。

第二步，估计先验概率和转移概率。

状态0的先验概率为，状态0出现的次数占比。

状态1的先验概率为，状态1出现的次数占比。

状态1转移到状态0的概率为，t时刻为状态1且t+1时刻为状态0出现的次数与t时刻为状态1次数的比值。

其余的以此内推。

第三步，取时间窗口为5，滚动每五分钟的状态转移序列出现的概率。

第四步，取市场收益率滚动5分钟状态转移序列出现的概率的5%分位数，作为阈值。

第五步，每个标的滚动5分钟转移序列出现的概率小于第四步得到的门限，就认为最后一个状态出现的时刻为异常时刻。

第六步，计算异常时刻的VWAP价格与其余时刻VWAP价格的比值。

**代码**

def process_single_day(self, idx):

    money = data['turnover'].shift(-4).dropna()
    vol = data['volume'].shift(-4).dropna()
    anomaly_vwap = (money * np.where(flag, 1.0, np.nan)
                    ).sum() / (vol * np.where(flag, 1.0, np.nan)).sum()
    norm_vwap = (money * np.where(flag, np.nan, 1.0)

第8行，计算收益率。

第9行，计算市场收益率。

第10行，调用cal\_prob方法计算状态转移序列的概率，这是该因子计算的核心。

第13-15行，计算的是异常时刻的占比。

第17-18行，将成交额和成交量往前移4分钟，即9:31的数据实际上是9:35的，这是为了和prob对齐。

第19-23行，就是计算聪明钱因子4.0版本了。

```python
@staticmethod
def cal_prob(data, mkt):
    mkt = mkt > 0
    prob = mkt.mean()
    data = data > 0
    state_prob = data * prob + (1 - data) * (1 - prob)
    transfer_11 = mkt.shift(-1)[mkt].mean()
    transfer_01 = mkt.shift(-1)[~mkt].mean()
    prob_list = [transfer_11, 1-transfer_11, transfer_01, 1-transfer_01]
    transfer_list = [data & data.shift(-1), data & (1-data.shift(-1)),
                     (1 - data) & data.shift(-1), ~data & (1 - data.shift(-1))]
    transfer_prob = 0
    for flag, prob in zip(transfer_list, prob_list):
        tmp_prob = flag * prob
        transfer_prob += tmp_prob
    transfer_prob = np.exp(np.log(transfer_prob).rolling(4).sum()).shift(-3)
    prob = state_prob * transfer_prob
    return prob.dropna()
```

接下来，该看核心代码了。

第3行，将市场收益率划分为两个状态。

第4-6行，计算每个标的每分钟先验概率的矩阵。这里先验概率的意思就是马尔科夫链的初始状态是0或者1的概率。

第7行，计算的是从状态1到状态1的概率，它与从状态1到状态0的概率之和为1。

第8行，计算的是从状态0到状态1的概率。

第9-15行，计算每个标的每分钟的状态转移概率矩阵，其中9:31的数据代表从9:31的状态转移到9:32的概率。

第16-18行，计算滚动5分钟状态转移序列出现的概率。

这里以9:31的数据进行简单说明，9:31代表从9:31到9:35这五分钟的状态转移序列出现的概率。它是9:31所处状态的先验概率\*(9:31到9:32的状态转移概率)\*(9:32到9:33的状态转移概率)\*(9:33到9:34的状态转移概率)\*(9:34到9:35的状态转移概率)。
