**计算步骤**

这个因子计算和极值是有关系的。

和上一篇文章相比，主要区别是将极值区分成了极大值和极小值。如果第一个极值点是极大值点，那么就用最后一个极小值点的价格和它来计算收益率；如果第一个极值点是极小值点，那么就用最后一个极大值点的介个和它来计算收益率。

这样得到的因子，就是笔者所说的日内极值反转因子。

下面，结合代码，深度了解一下这个因子。

代码

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close'])
    date_str = file_name.split('.')[0]
    close = data.to_dataframe('close')
    max_flag = ((close.diff() > 0) & (close.diff(-1) > 0))
    min_flag = ((close.diff() < 0) & (close.diff(-1) < 0))
    res = []
    for i in range(len(close.columns)):
        res.append(self.cal_factors(close.iloc[:, i], max_flag.iloc[:, i], min_flag.iloc[:, i]))
    res = pd.DataFrame(data=res, index=close.columns.tolist(), columns=['intraday_mom'])
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

这一次的代码同样分为两个部分，先来看看第一部分的代码。

第一部分的核心代码就是第8行和第9行。

第8行，计算的是极大值点。这里，close.diff()>0代表当前时刻的值大于前一时刻的值，close.diff(-1)>0表示当前时刻的值大于后一时刻的值。所以，这里得到的是极大值点的flag。

同理，第9行得到的就是极小值点了。

```python
def cal_factors(close, max_flag, min_flag):
    flag = pd.concat([max_flag, min_flag], axis=1)
    flag = flag.replace(False, np.nan)
    flag.reset_index(inplace=True, drop=True)
    flag = flag.dropna(how='all')
    if len(flag) == 0:
        return np.nan
    start = flag.iloc[[0]].index.tolist()[0]
    if flag.iloc[0, 0]:
        tmp_flag = flag.iloc[:, 1].dropna()
    else:
        tmp_flag = flag.iloc[:, 0].dropna()
    if len(tmp_flag) == 0:
        return np.nan
    end = tmp_flag.iloc[[-1]].index.tolist()[0]
    intraday_mom = close.iloc[start] / close.iloc[end] - 1
    return intraday_mom
```

接下来，就是第二部分代码了。

这部分代码笔者没有选择用for循环的方式来实现，而是将极大值点和极小值点的flag拼接为了一个dataframe，对应代码第2行。

第3行，将False转换为nan，然后在第5行代码中将整个一行都为nan的去除。这样，第一行的索引对应的一定是第一个极值点。

在两者之间的第4行，是为了将索引转换为数值，这样接下来计算收益率的时候可以用iloc而不是用loc了，速度更快。

第9~12行，根据第一个极值点的属性（是极大值还是极小值），来确定最后一个极值点该选什么。如果flag.iloc[0, 0]为True则代表第一个点为极大值点，否则为极小值点。

剩下的代码就是根据极值点的索引来计算收益了，其中还包含了一些异常情况的处理。如果数据没有极值点或者只有一个极值点，那么将返回nan。

**总结**

这个因子从IC上的表现来看，用均值和标准差合成的差异是不大的。同时，这个因子的IC绝对值是比上一篇文章中介绍的因子的IC绝对值略微高那么一点点的。

由于，从分层回测的结果来看，标准差合成的因子效果更好，所以笔者上面的因子评价结果展示的是用标准差合成的。
