上周的时候，笔者复现了兴证金工首席郑兆磊老师的研报。在这篇研报中，他用高斯混合分布将分钟级收益率划分为了跳价期和震荡期，然后设计了一系列的因子。![图片](images/2025-04-02_用高斯混合分布将分钟级收益率划分为震荡期、下跌期和上涨期之和，看起来比跳价期和震荡期要好！/001.jpg)

在笔者的复现过程中，笔者发现IC和分层回测表现最好的是跳价期的波动这一因子。

在之后的这段时间里，笔者一直在思考如何在这个基础上改进，提出一个真正的属于自己的因子。

上周日，笔者尝试了将高斯混合模型换成了k-means聚类算法，然后通过该方法想出一个能拥有几百个因子的套路。但是，从各种表现来看，它们并没有超越跳价期的波动这一因子。

于是，继续冥想，然后就有了本文的思路，通过高斯混合分布将分钟级别的收益率划分为震荡期、下跌期和上涨期三个阶段，具体的实现如下。

```python
def gmm_fit(data):
    flag = ~np.isnan(data)
    arr = data[flag]
    gmm = GaussianMixture(n_components=3, covariance_type='full')
    gmm.fit(arr.reshape(-1, 1))
    keys = gmm.weights_
    mu = gmm.means_.reshape(-1)
    sigma = np.sqrt(gmm.covariances_).reshape(-1)

    fluctuation_idx = np.argmax(keys)
    res = [keys[fluctuation_idx], mu[fluctuation_idx], sigma[fluctuation_idx]]

    up_idx = np.argmax(mu)
    if mu[up_idx] < 0 or mu[up_idx] - mu[fluctuation_idx] <= 1e-4:
        mu[up_idx] = 0
    res += [keys[up_idx], mu[up_idx], sigma[up_idx]]

    down_idx = np.argmin(mu)
    if mu[down_idx] > 0 or mu[fluctuation_idx] - mu[down_idx] <= 1e-4:
        mu[down_idx] = 0
    res += [keys[down_idx], mu[down_idx], sigma[down_idx]]
    return res
```

前8行代码，和之前划分为跳价期和震荡期是一样的。

第10-11行，通过最大权重获取震荡期的索引，然后得到该时间对应的权重、均值和方差。

第13-16行，通过最大均值获取上升期的索引。这里需要将最大均值和震荡期均值比较，如果最大均值小于0或者和震荡期均值接近的话，就将上升期的均值置为0。其实，概率应该也置为0，然后将这部分概率算作震荡期的。纠结了一下，然后考虑到满足条件的时候概率都很低了，影响应该不是太大。各位读者可以尝试将概率也置为0试一试。

第18-21行，逻辑和第13-16行是一致的。

从因子数量来说，除了每个时期的权重、均值和方差外，还可以增加一个上行期涨幅与下行期跌幅之和，计算代码如下。

```python
data = (data['up_mean'] * data['up_prob'] * 240 +
                                 data['down_mean'] * data['down_prob'] * 240)
```
