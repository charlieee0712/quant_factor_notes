**遗憾规避**

本文，笔者要复现因子的思想来自国金证券高智威老师在2023年2月27日发布的研报《Alpha掘金系列之四：基于逐笔成交数据的遗憾规避因子》。

遗憾的是，笔者没有逐笔成交数据，所以只能在理解其核心思想的基础上，对其进行魔改，使其能用1分钟的数据来进行计算。

在研报中，主要是有四个因子，分别是买入浮亏占比因子（HCVOL）、卖出反弹占比因子（LCVOL）、买入浮亏偏离因子（HCP）和卖出反弹偏离因子（LCP）。

关于这四个因子的思路，研报中是这样描述的。

> 高于收盘价买入成交量占比越高，则投资者当天买入情绪较高，且浮亏占比较高，未来抛压较低，有着更高的预期收益。同理，高于收盘价买入的价格相较于收盘价偏离幅度越大，则投资者浮亏现象越严重，也会有更低的抛压，带来更高的预期收益。而低于收盘价卖出成交量占比越高、或价格相较于收盘价向下偏离越大的股票，投资者受到卖出行为的影响，不愿轻易再次买回，因此此类股票未来买入动力较低，有更低的预期收益。
>
> 根据以上思路，HCVOL 和 HCP 均应为正向因子，LCVOL 应为负向因子，而 LCP 由于所计算价格偏离为负数，此因子值越低，后续买入动力越弱，预期收益越低，应为正向因子。

**计算步骤和代码**

、

买入浮亏占比因子是买入价格大于收盘价的成交量与总成交量的比值。

卖出反弹占比因子是卖出价格小于收盘价的成交量与总成交量的比值。

买入浮亏偏离因子是所有买入的均价相比收盘价的偏离程度。

卖出反弹偏离因子是所有卖出的均价相比收盘价的偏离程度。

由于笔者没有逐笔数据，所以只能用分钟数据对上述因子进行魔改，魔改的主要思想来自于长江证券覃川桃老师的研报，具体内容在[分布估计下的主动买入占比，桃子姐的因子居然在这本书中提到过！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247492705&idx=1&sn=503a5c42c1a673e9e959fc20792c9669&scene=21#wechat_redirect)这篇文章中。

**计算步骤**

第一步，通过t分钟收盘价与t-1分钟收盘价的差分来判断该分钟属于买入还是卖出，如果差分大于0则是买入，否则是卖出。这是一个非常粗糙的判断。

第二步，将买入分钟的均价与最后的收盘价比较，小于收盘价就是买入浮亏时刻；同理，将卖出分钟的均价与最后的收盘价比较，大于收盘价就是卖出反弹时刻。

第三步，用买入浮亏时刻对应的分钟数据计算买入浮亏占比和买入浮亏偏离，用卖出反弹时刻对应的分钟数据计算卖出反弹占比和卖出反弹偏离。

**代码**

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['turnover', 'close', 'volume'])
    date_str = file_name.split('.')[0]
    cur = pd.to_datetime(date_str) + timedelta(hours=15)
    if cur > pd.to_datetime('2026-01-01') or cur < pd.to_datetime('2005-01-01'):
        return pd.DataFrame(columns=['hcvol', 'lcvol', 'hcp', 'lcp'])
    factor = self.factor.loc[cur]
    factor = factor.reindex(data.codes)
    close = data.to_dataframe('close') * factor.values.reshape(1, -1)
    if self.last_close is not None:
        close = pd.concat([self.last_close, close])
        self.last_close = close.iloc[[-1]]
    flag_up = close.diff() > 0
    flag_dowm = close.diff() < 0
    money = data.to_dataframe('turnover')
    vol = data.to_dataframe('volume')
    tot_vol = vol.sum()
    vwap = money / vol
    end_close = data.to_dataframe('close').iloc[-1]
    buy_down = vwap * np.where(flag_up, 1.0, np.nan) > end_close
    sell_up = vwap * np.where(flag_dowm, 1.0, np.nan) < end_close
    hcvol = (buy_down * vol).sum()
    lcvol = (sell_up * vol).sum()
    hcp = ((buy_down * money).sum() / hcvol) / end_close - 1
    lcp = ((sell_up * money).sum() / lcvol) / end_close - 1
    res = pd.concat([hcvol / tot_vol, lcvol/ tot_vol, hcp, lcp], axis=1)
    res.columns = ['hcvol', 'lcvol', 'hcp', 'lcp']
    res['datetime'] = cur
    return res
```

前15行，读取数据。

这里有两点需要注意，第一个是读取复权因子，因为笔者的复权数据是日频的，时间区间和分钟频不一致，所以在第8行进行了一个保护性判断。

第13-15行，保存了上个交易日最的收盘价，这样每个交易日的第一天就不会缺数据了。但是，这样处理的话就不能并行加速了。还好，这个因子的计算并不是太慢。

第16行，判断主动买入时间。

第17行，判断主动卖出时间。

第18-21行，获取成交额和成交量，并计算均价。

第22行，获取每日的收盘价，这里的收盘价没有乘复权因子，因为均价也没有乘复权因子。

第23-24行，计算买入浮亏时刻和卖出反弹时刻。

第25-28行，计算四个因子。

不过，有一点和研报中的因子不一样，在研报中hcvol、hcp和lcp都应该是正向因子，而lcvol则应该是反向因子。

但是，从相关性来看，lcp因子的方向和其他三个因子相反。笔者测试的时候发现，只有lcp一个因子是正向因子（都用均值合成的情况下，如果都用标准差合成则都是复向因子），而且是IC表现最好的正向因子。
