同时，为了进行对比，笔者又去掉了市值，再去掉了行业，看看这一通略微有一点点复杂的操作会带来什么样的提升。在[用行业和市值计算的注意力溢出，相比注意力的提升居然在这里！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247490403&idx=1&sn=9d6ec36fa55f57c474f5f636c562b920&scene=21#wechat_redirect)文章中，笔者发现这种操作会提升因子在分层回测上的表现，分组之间的区分度大幅提升，相交的情况减少。

受到了它们的启发，笔者想将这个思想进行推广，看在其他因子上是不是也能产生同样的效果。

由于这个操作是对收益率进行的，所以笔者首先想到的也是在用到收益率均值的场景来验证这一想法。就这样，选择了自信溢出这个因子，其原始因子在[一个直观上感觉离谱的因子，稍微改造一下居然每年的IC绝对值都超过了0.08！](https://mp.weixin.qq.com/s?__biz=MzkwNjYzMTEyMg==&mid=2247490984&idx=1&sn=79d344fe276593300a37827881846fa4&scene=21#wechat_redirect)这篇文章中。

**计算步骤和代码**

值得一提的是，笔者在实践中发现，在考虑了行业和市值的情况下，在将相关性也考虑进来，分层回测的效果会更好一点点。

所以，在这里，笔者将介绍两种步骤，不考虑相关性的和考虑相关性的。

**1**

**不考虑相关性的计算步骤**

不考虑相关性的计算步骤比较简单，就是对标的先按照行业分类，再按照市值分为大中小三类，分别计算每一类标的收益的平均值。

**2**

**考虑相关性的计算步骤**

考虑相关性的方式，需要用过去252个交易日的收益率计算标的之间的相关性，然后按照行业市值分类之后，用相关性加权来为每一个标的计算参考收益率，也就是回归时候的x。

这里需要注意的是，相关性加权的收益率有两种计算方式，第一种把自身也考虑了进去，第二种则是不考虑自身。

笔者的测试选择的是第二种方式。

def __call__(self):

    start = self.ind_data.index.tolist()[0]

                                    codes=codes, start=start, lag=4).to_dataframes()
    close = data['close'] * data['factor']
    vol = data['volume'] != 0
    close = close * np.where(vol, 1.0, np.nan)
    self.rtn = close.pct_change().iloc[1:]
    res = Parallel(n_jobs=16, verbose=10)(
        delayed(self.process_single_day)(idx)
        for idx in range(252, len(self.rtn))

    res.reset_index(inplace=True)
    res = pd.melt(res, id_vars='datetime', var_name='code', value_name='rnbr_ret')
    res.to_parquet('./rnbr_ret.parquet')

第一部分代码，注意是读取数据、数据预处理和数据后处理。这一块的内容和本文引用的文章中的代码是类似的，所以就不赘诉了，仅仅贴出来给各位参考。

```python
def process_single_day(self, idx):
    rtn = self.rtn.iloc[idx-252:idx]
    day = rtn.iloc[[-1]].index.tolist()[0]
    last_rtn = rtn.iloc[-1].dropna()
    codes = last_rtn.index.tolist()
    rtn = rtn.reindex(columns=codes)

    corr = rtn.corr()
    corr_values = corr.values
    np.fill_diagonal(corr_values, np.nan)
    corr = pd.DataFrame(corr_values, index=corr.index, columns=corr.columns)
    
    df = pd.concat([last_rtn, self.ind_data.iloc[idx-1], self.market_cap.iloc[idx-1]], axis=1)
    df.columns = ['rtn', 'ind', 'cap']
    df = df.join(corr).dropna(subset=['rtn'])
    new_series = df.groupby('ind', as_index=False, group_keys=False).apply(self.__cal_factors__)
    
    codes = new_series.index.tolist()
    x = new_series.values
    idx = ~np.isnan(x)
    x = x[idx].reshape(-1, 1)
    y = last_rtn.reindex(codes).values.reshape(-1, 1)[idx, :]
    lr = LinearRegression()
    lr.fit(x, y)
    resi = y - lr.predict(x)
    res = pd.Series(resi.reshape(-1), index=np.array(codes)[idx])
    res.name = day
    return res
```

第二部分代码就是整个因子计算的核心了。

这一部分代码也可以分为四块。

第一块是2-6行，第2行获取过去252个交易日的收益率数据；第3行获取计算出来的因子所属的交易日；第4行获取最新一个交易日的收益数据，并去掉为nan的；第5-6行，只保留最新一个交易日收益率不为nan的标的的过去252个交易日的收益率数据。

第二块是8-11行，这里计算了收益率的相关性，然后将对角线的元素置为nan，也就是不考虑自身了。因为用np.fill\_diagonal之后，dataframe会变成numpy数组，所以需要重新把它变成dataframe。

第三块代码是13-16行，将收益率、行业和市值以及相关性的信息拼接为一个dataframe。这里需要注意的是第2行取数据的时候是取到了idx-1，所以再取行业和市值数据的时候也是用idx-1来取的。

第四块代码是18-28行，就是执行线性回归然后计算残差了，这块代码之前也详细介绍过，就不细说了。

```python
@staticmethod
def __cal_factors__(group):
    group.sort_values(by='cap', inplace=True)
    start = 0
    res = []
    for q in [0.3, 0.7, 1]:
        end = int(len(group) * q)
        tmp_group = group.iloc[start:end]
        cols = tmp_group.index.tolist()
        tmp_rtn = tmp_group['rtn'].values.reshape(-1, 1)
        weights = tmp_group[cols]
        weights = weights.div(weights.sum(), axis=1)
        tmp = np.nansum(tmp_rtn * weights, axis=0)
        res += list(tmp)
        start = end
    res = pd.Series(data=res, index=group.index.tolist())
    return res
```

第三部分代码，也是在之前代码的基础上进行微微的修改。主要是第9-14行，第9行获取标的信息，用于第11行获取相关性对应的列；第10行获取收益率；第11-12行，计算加权均值的权重；第13行，计算加权收益率；第14行，保存结果。
