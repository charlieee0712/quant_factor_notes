**知情交易**

自从方正证券的曹老师通过FFT变换来识别VWAP或者TWAP交易（通常也被称作知情交易者）之后，笔者似乎对通过分钟数据来寻找VWAP或者TWAP交易的痕迹走火入魔了。

但是，自己多次尝试都不太成功。

眼看着就要走火入魔了，这时候笔者想到了AI，于是让豆老师推荐一个利用分钟级别的数据来识别知情交易者参与的概率。

于是，就有了这个因子。豆老师表示，噪声交易者主导的标的，其换手率和波动率的相关性很低，而知情交易者主导的市场换手率和波动率的相关性会变高，甚至可能超过0.7。

在这种情况下，就有了本文的因子。

没错，就是换手率和波动率的相关性，豆老师推荐是计算滚动60分钟的换手率和波动率的相关性。

这一次，笔者没听劝，而是分别计算了滚动5分钟、滚动30分钟和滚动60分钟的相关性。

第一段代码，是在初始化的时候读取了一下流通股本的数据，方便计算滚动n分钟的换手率。

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]
    cur = pd.to_datetime(date_str) + timedelta(hours=15)
    if idx < 243:
        return pd.DataFrame(columns=['corr_5', 'corr_30', 'corr_60', 'datetime'])
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'close', 'open'])
    cap = self.cap.iloc[idx - 243].reindex(data.codes)
    cap = cap.values.reshape(1, -1)
    rtn = data.to_dataframe('close') / data.to_dataframe('open') - 1
    vol = data.to_dataframe('volume')
    res = []
    for i in [5, 30, 60]:
        res.append((vol.rolling(i).sum() / cap).corrwith(rtn.rolling(i).std()))
    res = pd.concat(res, axis=1)
    res.columns = ['corr_5', 'corr_30', 'corr_60']
    res['datetime'] = cur
    return res
```

这段代码也很简单，唯一需要注意的是第6-7行。

因为笔者的日频数据是从2005年开始的，而分钟数据是从2004年开始的，这两个数据之间有个gap。通过观察数据，知道了2004年有243个交易日，那么2005年的第一个交易日就是从索引243开始。所以，当索引小于243时，直接返回空的数据。

其他的代码就不需要什么特别的说明了。
