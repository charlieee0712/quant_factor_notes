这个因子，在去年的时候，笔者在日频的数据上强行复现过一次。

该因子来自长江证券首席分析师覃川桃老师在2020年发布的研报《高频因子(八）：高位成交因子——从量价匹配说起 》

为什么用重新拿出来说呢？有两个原因。

第一，想在分钟数据上看看效果。

第二，想用这个优化一下Amihud非流动性因子，看看会不会有不一样的效果。这里直接说结果吧，优化失败了，IC不高，分层回测也不是很好。

那么，这个因子是如何计算的呢？

对于分钟级别的k线，用t时刻的成交量/当日的总成交量得到权重，用权重乘以收盘价并求和，得到一个加权收盘价。最后，用加权收盘价/当日收盘价的均值即可。

```python
def process_single_day(self, idx):
    file_name = self.files[idx]
    date_str = file_name.split('.')[0]

    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['close', 'volume'])

    res = (data.data[:, 0, :] * data.data[:, 1, :] / np.nansum(data.data[:, 1, :], axis=0, keepdims=True))
    res = np.nansum(res, axis=0) / np.nanmean(data.data[:, 0, :], axis=0)
    res = pd.Series(data=res, index=data.codes, name=pd.to_datetime(date_str) + timedelta(hours=15))
    return res
```

第8行，计算加权收盘价。

第9行，计算加权收盘价与收盘价均值的比值。

这个因子是想刻画标的在高位成交密集水平，与未来收益率负相关。因子值越高，标的在高位成交越密集，因交易中的羊群效应，推动股价高估，未来收益率会回调。

根据这个说明来看，这个因子应该对过去21个交易日分钟级别的数据来计算。但是，笔者的代码是每天单独计算的，在评价的时候采取了三种不同的方法，第一种，过去21个交易日求均值；第二种，过去21个交易日求标准差；第三种，直接不做任何处理。

在三种方法都尝试了一下之后，发现标准差的效果看起来好一点点。所以，就展示了一下标准差的评价结果。只不过，用标准差的话似乎与这个因子设计的初衷不相符了。

标准差越大，证明在过去21个交易日中，标的有的时候在高位成交，有的时候不是，而且这两种状态出现的情况相当。标准差越低的话就意味着，标的大部分时间都在高位成交，或者大部分时间都不在高位成交。

这个因子通过测试是与未来收益负相关的。理论上，如果标准差低，然后如果都是高位成交的话，按照研报中的说明，过去21天该标的可能都是在高位成交的，那么它未来收益率应该会下降。这似乎可以有一个修正的思路。本文我们暂且按下不表，等笔者改进一下试一试。（文末有一个简单的改进，效果还可以！）

总结：这个因子在IC上的表现还算是不错的，每一年的IC绝对值都超过了0.06，除了2018年和2024年（30分钟频率开始，2019年的IC绝对值也低于0.08了），其他年份的IC绝对值甚至超过了0.08。同时，在这个回测区间，该因子只有9个月的IC与其他月份反向，方向很稳定。但是，分层回测方面的表现就相对一般了，相交很多，在回测区间也没有表现出很好的单调性。

另外，该因子在不同频率的K线上的表现没有实质性差异。

在写到这个总结的时候，笔者对前面提到的修正进行了一些简单的思考。

标准差越低，未来收益越高，这个是测试测出来的。

均值越高，未来收益率越低，这个是研报中关于因子原理描述得出的一个结论。

同时，标准差低的时候，可能有两种情况，一种是均值高，一种是均值低。构造的目标就是将这两种情况区分出来。于是，尝试了一下标准差/均值和均值/标准差的情况，发现还真的可以。

其中，标准差/均值叫做变异系数，这个和直接用标准差区别不大。区别大的是它的倒数，也就是均值除以标准差。
