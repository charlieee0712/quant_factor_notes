一阶残差的自相关性代码还没有跑完，所以本文，笔者将复现长江证券在2020年10月12日发布的研报《基础因子研究（十四）：高频因子（九），高频波动中的时间序列信息》中的因子。

这篇研报中也有很多的因子，首先复现的是差分标准差因子。

为什么选择这个因子呢？因为它的计算简单，而且很快，适合在周末不想卷又不想断更的懒人。

这个因子的计算，虽然有三步，但是每一步都很简单。

第一步，计算因子的差分，可以是成交量，也可以是成交额，甚至可以是振幅。

第二步：计算日内差分的标准差然后除以原始因子的均值。

第三步：对过去21个交易日求均值。

这里，笔者还是按照之前的思路，用成交量和成交额分别计算，然后还将两者的差分作为因子，具体代码如下：

```python
def process_single_day(self, idx):
    # 加载当日分钟数据
    file_name = self.files[idx]
    full_path = os.path.join(self.file_pth, file_name)
    data = BaseDataLoader.load_data(full_path, fields=['volume', 'turnover'])
    date_str = file_name.split('.')[0]

    vol = data.to_dataframe('volume')
    money = data.to_dataframe('turnover')
    vol = (vol.diff()).div(vol.mean(axis=0), axis=1)
    money = (money.diff()).div(money.mean(axis=0), axis=1)
    diff = money - vol

    res = [vol.std() / vol.mean(), money.std() / money.mean(), diff.std(), diff.mean()]
    res = pd.concat(res, axis=1)
    res.columns = ['vol', 'money', 'diff_std', 'diff_mean']
    res['diff'] = res['money'] - res['vol']
    res['datetime'] = pd.to_datetime(date_str) + timedelta(hours=15)
    res.index.name = 'code'
    return res.reset_index()
```

核心代码是第10-11行。

这里需要注意的是，对于二者的差分，笔者采用了两种计算方式，第一种是每分钟因子的差分，然后分别取均值和标准差。第二种是，日频的因子直接作差。

diff

这个因子就不一样了，如果是用均值合成的话，就不太行了。IC绝对值最高的一年也不超过0.015。

到这里，笔者觉得挺正常的。毕竟，之前类似的因子也都是这样的，用标准差合成月度因子效果就会出现质的改善。但是，用标准差合成的因子和上面一个因子的相关性可能就高了。于是，正当笔者准备在进行一次相关性检验的时候，打脸的事情发生了。

这个因子无论是用均值合成还是用标准差合成，都不太行。

但是，笔者不愿意就这样认输。

所以又试了以下diff\_mean这个因子。

diff\_mean

这个因子用标准差合成月度因子之后，虽然IC绝对值最高也不超过0.1，但是它有5年的绝对值是超过0.08的，看起来比diff\_std更加稳定。

至于用均值合成的因子，其IC绝对值只有两年是超过0.05的。
