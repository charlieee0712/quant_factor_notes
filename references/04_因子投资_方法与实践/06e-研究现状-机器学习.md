> 石川/刘洋溢/连祥斌《因子投资：方法与实践》· 视觉精读(PDF 物理页 p.281–290) · 生成 2026-06-30

## 本块讲什么

§6.8 整节系统综述"机器学习(machine learning, ML)能为因子投资做什么、坑在哪"。脉络是：先给 ML 在因子投资中的**定位**（核心 = 预测 + 特征选择），再按"有监督 → 无监督"两条线展开——

- **6.8.1 线性模型**：OLS 之上的四类拓展（稳健回归 / 惩罚回归 / 降维 / 广义线性），外加混合预测(forecast combination)。
- **6.8.2 非线性模型**：树模型(决策树→回归树→boosting/bagging→GBDT/XGBoost/随机森林)、支持向量机(SVM)、神经网络(NN→DFN/RNN/LSTM)。
- **6.8.3 模型评估与实证研究**：样本外 R²(R²_OOS)、Diebold–Mariano 检验、Gu et al.(2020) 60 年美股大实证、A股对照结论。
- **6.8.4 主成分分析和因子选择**：无监督学习路线——隐性因子模型 + PCA / IPCA / RP-PCA 同时估因子暴露与溢价。
- **6.8.5 机器学习的问题**：两大软肋——黑箱(可解释性)与过拟合，及对应缓解手段；作者立场"结合而非取代"。

一句话立意：**ML 不是"锦上添花"，而是因子投资已经"必要"的工具**——经典排序法 + 线性回归在高维预测变量下力不从心，少数经典因子也解释不了截面差异；但 ML"虽好也非万能良药"。

## 方法论要点（重点·展开）

### A. ML 在因子投资里到底做什么（定位，决定后面一切）

- **定义(Gu et al. 2020)**：ML = "一系列服务于统计预测的高维模型 + 用于模型选择和防过拟合的正则化方法 + 对大量候选模型设定有效筛选的算法"。在因子投资场景，**核心是预测**。
- **与经典统计的根本不同**：经典统计重推断(inference)，ML 天生为预测而生。
- **两大功能**：
  1. **预测(prediction)**——个股收益预测 = **截面问题**（公司特征 ↔ 预期收益的横截面关系，正是因子投资关注点）；市场整体预测 = **时序问题**。
  2. **特征选择(feature selection)**——训练时算法给不同解释变量赋不同权重，于是"顺便"学到哪些特征对预测更重要。
- **为什么必须用 ML**：经典组合排序法 / 线性回归处理大量预测变量时有各种限制；且仅靠少数经典因子无法解释股票预期收益的截面差异。这是全节的动机底色。

### B. 线性模型——OLS 之上的四类拓展（"先把线性做对"）

多因子模型本就假设因子与收益线性，OLS 是最简单解。书把拓展归四类：

1. **稳健回归(robust regression)**，两条支线：
   - **加权回归**：OLS 对每个样本点等权（最小化残差平方和）；加权回归赋不同权重。Fama–MacBeth 回归早有应用；面板估计公司特征对收益影响时，**用当期股票数作权重**——某期股票越多，结果越可靠。
   - **稳健损失（适应肥尾收益）**：把目标函数设为 **Huber 稳健误差函数**[式 6.30+6.31]。直觉：残差小用平方损失、残差大改用线性损失，从而压低离群点（肥尾收益）对估计的拉扯。
2. **惩罚回归(penalized regression)**：Ridge(加 L2 罚) / LASSO(加 L1 罚) / Elastic Net(同时 L1+L2)。相对 OLS 额外加针对高维数据的惩罚项；**同时起到筛选有效预测特征的作用**。（L2/L1 退化：参数为向量时 L2 = 平方项、L1 = 绝对值。）
3. **降维方法(dimension reduction)**：主成分回归(PCR) 与偏最小二乘(PLS)。显著降低问题维度 → 更稳健估计。例：Chen et al.(2019) 用 PLS 从 12 个常见情绪代理指标中提取信息、更好刻画投资者情绪。
4. **广义线性模型(GLM)**：介于标准线性与非线性之间。
   - 加公司特征**高次项**：典型 = Barra 非线性规模因子(市值的三次方)；Gu et al.(2020) 用公司特征的 **k 项样条函数**作更一般设定。
   - **离散因变量建模**：只关心下月超额收益是否为正 → 0/1 因变量 → **logistic 回归**；关心收益**排序分组**(分 G 组)而非具体值 → **multinomial logistic 回归**。
5. **混合预测(forecast combination)**：并非具体算法，而是**对一系列算法的预测取平均**。思想：单一算法不会总有效，取均值平滑不同算法误差 → 更有效。Rapach et al.(2010) 用于市场溢价(经典，RFS 2010、2017 最多阅读)、Han et al.(2018) 系统讨论 LASSO/Elastic Net/混合预测在收益截面的应用。属集成(ensemble)思想。

### C. 非线性模型谱系（"线性只是真实模型的一阶近似"）

- **动机**：线性难刻画解释变量间**相互影响(交互)**；理论上可加交互项，但变量数一多交互项**爆炸式增长**，缺先验难选加哪些 → 非线性模型登场。

**1. 树模型**
- **决策树**→重要特征作根节点，按特征逐层分类；非参、计算密集，但算力进步后可处理大样本+多特征，**能挖掘特征间相互作用**、解释性较强、不要求特征独立。
- **回归树**：因变量连续(预期收益)时，本质仍是分类，只是给同类样本一个**公共观测值**(用每类**收益均值**作该类预期收益)。例：先按市值前/后 50% 分，后 50% 再按价值 30% 分位细分为三类。
- **两大优于线性之处**：(1) **L 层树天然含 L-1 层交互效应**（线性不具备）；(2) **不受解释变量单调变换影响**，能很好容纳非线性。
- **最大瓶颈 = 过拟合**（灵活性的代价）→ 需正则化：boosting 与 random forest。
  - **boosting**：框架算法，一堆高度简化的分类树迭代生成多个**弱分类器**，组合成**强分类器**（与混合预测同属集成）。谱系：AdaBoost(按上次预测误差更新样本权重，错的权重升高) → **GBDT**(梯度提升树：弱分类器只用回归树 + **对上一步的拟合残差建模**，前向分步；例：第一步按市值预测 3.0%、真实 1.2%，第二步用价值特征去拟合残差 1.8%) → **XGBoost**(改进 GBDT 的稀疏数据短板，数据挖掘大赛表现优异)。
  - **bagging**：与 boosting 相对——**可并行**多次训练，每次用 **bootstrap 抽子样本**训练、最后取平均；分类问题的典型 = **随机森林**。
  - **关键区分**：boosting 顺序迭代(后一步依赖前一步残差)；bagging 并行+抽样平均。

**2. 支持向量机(SVM)**——反其道而行：不降维，用**核函数(kernel)升维**，在高维空间找**最大间隔超平面**做线性分割，对应原空间的**非线性决策边界**；升维不增计算量，故高效。深度学习/XGBoost 流行前曾是最重要算法之一。

**3. 神经网络(NN)**——深度学习基础，"最有效"的 ML 算法。
- 结构：输入层(input)→隐藏层(hidden)→输出层(output)。若预期收益与特征间的预测模型可用一个**光滑函数**表达，NN 即可有效逼近。
- 训练：常用最小化预测误差的 **L2 范数惩罚**估计权重；**优势 = 每步可同时更新所有参数**（树方法做不到）。但高度非线性 + 巨量参数 → 计算复杂、易过拟合 → 用**随机梯度下降(SGD)**(牺牲精度换效率) + 多种正则化(**学习率收缩 / 早停 early stopping / 批标准化 batch normalization / 集成**)。
- 发展：DFN(深度前馈) / RNN(循环) / LSTM(长短期记忆)。
- 问题集中在**参数规模、模型复杂度、可解释性**。

### D. 模型评估（可操作，回测必备）

- **样本外 R²(R²_OOS)**[式 6.32]：衡量"相对历史均值预测，模型样本外 MSE 是否更小"；模型不如历史均值时 **R²_OOS 为负**。Welch–Goyal(2008)、Campbell–Thompson(2008) 用它检验诸多变量对**市场总收益**的预测力。
- **关键陷阱与正确基准**：预测**个股**收益时，历史均值预测**极糟糕** → Gu et al.(2020) 主张用 **"零"替代历史均值作基准**[式 6.33，分母变为 ΣR²]。实证：若改用历史均值基准，所有候选模型 R²_OOS 会**虚高约 3%**；而一般 R²_OOS 超 **0.5%** 就算有价值——3% 的虚高足以得出**误导性结论**。
- **模型两两比较 = DM 检验**[式 6.34，源自 Diebold–Mariano 2002]：构造两模型样本外 MSE 之差 d₁₂ 的标准化统计量；**DM₁₂ 越大 → 模型1 越差，越小 → 模型1 越好**。即比较模型不靠肉眼比 R²，而靠有标准误的统计量。

### E. PCA / 隐性因子——无监督学习做因子选择（6.8.4 重点）

- **动机（经典 FM 回归的病）**：Fama–MacBeth 等经典法**必须先明确指定因子结构**才能有效估计，且易受**遗漏变量、测量误差**影响。但越来越多研究指出"**我们其实不知道真实定价因子是什么**" → 把真实因子视作**隐性因子(latent factors)**，用降维**同时估因子暴露与因子溢价**。
- **隐性因子模型**[式 6.35]：R^e_{it}=β'_i λ_t+ε_{it}，形似普通多因子模型，**特别之处 = 真实因子不可观测**（λ 未知 → β 也未知）。
- **Giglio–Xiu(2019)（开创性）**：取资产收益**协方差矩阵主成分**估溢价/暴露。两条计量性质支撑：(1) 线性多因子模型的**旋转不变性**——只观测到隐性因子的某满秩变换也能估可观测因子溢价；(2) 隐性因子足够强时，**PCA 总能复原因子空间的某旋转(Bai 2003)**。结论：真实因子不可观测，仍能**准确估可观测因子溢价**。实证亮点：解决 FM 回归对控制变量的高度依赖——动量因子在"不控制其他因子" vs "控制 FF3"下溢价**符号竟相反且都显著**，PCA 法则给出稳定满意估计。
- **Rapach–Zhou(2019)**：用**稀疏 PCA** 从 120 个宏观变量提 10 个稀疏主成分(可解释性好，近似债券名义收益率/通胀率/产出率) → 用 GX 法估溢价 → 构建**稀疏宏观四因子模型**，解释力比肩 Hou–Xue–Zhang 四因子与 FF5。
- **Kelly et al.(2019) IPCA（动态条件定价）**：经典 PCA 只适**静态**模型；引入**大量公司特征作工具变量(instrumental variable)**，把**因子暴露参数化为公司特征的函数** → 解决动态条件资产定价。两点启发：公司特征与因子暴露密切相关(可作 IV)；公司特征随时间变化(时序法难处理)。可视作**有约束的 FM 回归**(因子数=特征数时退化为一般 FM)。实证震撼：**6 个 IPCA 因子的样本外切线组合 Sharpe 高达 4.05**(vs FF5+动量共 6 因子仅 1.37)；真正起作用的是**公司特征的动态变化**。可解释性：PC1≈价值/杠杆、PC2≈市场、PC3/PC4≈动量/短期反转。
- **Lettau–Pelger(2020) RP-PCA**：上述 PCA 只用**二阶矩(协方差)**信息，丢掉**一阶矩**（因子与收益的截面关系）；在经典 PCA 目标函数中**加入代表一阶矩的项** → 风险溢价 PCA。实证：绝大多数情况优于 PCA / FF5，5 个 RP-PCA 因子能反映系统性风险 + 解释截面差异，且各有经济学基础。
- **两条因子投资落地思路 + 警示**：(1) 直接用 PCA 提主成分因子→倒推因子组合/公司特征权重建组合；(2) 用 PCA 因子对公司特征/经典因子的暴露**映射回经典因子**再配置。**警示**：现实**卖空约束**使按上述方法建组合**更复杂、可投资性更低**；该路线仍是"最近几年新趋势"、**不成熟**，**定位是对经典方法的有效改进而非取代**。

### F. 机器学习的两大问题（6.8.5，全节落点）

**(1) 黑箱 / 可解释性**：NN 等内部极复杂，黑箱性质使人难懂"特征 ↔ 未来收益"的关系。**缓解**：Dixon–Halperin(2019) 通过算**每个特征对最终输出的影响程度**来解释；正确姿势 = 先找到有显著预测力的模型 → 再考察哪些特征显著 → 梳理特征与收益的逻辑关联。理解 ML 发现的规律 = 研究的"基石"，反哺经济逻辑。

**(2) 过拟合，多重根源（逐条记，全是回测坑）**：
- **路径唯一**：真实资产价格路径只有一条，基于它反复训练**本身就易过拟合**，易踏入 Harvey et al.(2016) 的 **p-hacking** 陷阱。
- **WFR 不根治**：前向回测(walk-forward backtesting) **只能避未来数据(look-ahead)，不能根除过拟合**。
- **样本太小**：因子研究常用**月度**收益，最长美股较完整数据自 **1962** 起 ≈ **700 个月**——对 ML 而言样本过小。
- **CV 失效**：金融数据的**序列自相关 + 异方差**使训练集信息**泄漏到测试集**，导致交叉验证(cross-validation)失效。
- **Sharpe 高估**：给定收益分布，**预期最高夏普与波动率正相关** → 高波动资产在单次历史回测中**反而易得更高 Sharpe** → 单纯历史回测会**高估因子表现**。
- **对策**：模拟生成**多条更长周期**价格路径、分析不同场景；确保训练/测试集在**时间区间上不交叠**；用**平减夏普(deflated Sharpe ratio)** 部分解决 Sharpe 高估。

**作者立场（结论句）**：ML 应用尚处**早期**；能否取代经典多因子模型成主流"待进一步考察"；ML 终究是**数据模型方法，要靠对业务领域知识的理解才能发挥作用**——**最佳路径是与已有方法结合，而非取而代之**。

## 关键公式 · 定义 · 判定标准

- **式(6.30) Huber 目标函数**：L(θ)=(1/NT)·Σ_{i=1}^N Σ_{t=1}^T h( R_{i,t+1} − g(z_{it};θ) ; ξ)。N/T = 股票数/期数，R_{i,t+1}=股 i 在 t+1 真实收益，z_{it}=截至 t 可用的预测变量向量，g(z_{it};θ)=对 t+1 期收益的预测。
- **式(6.31) Huber 核**：h(x;ξ)= x²（若 |x|≤ξ）； 2ξ|x| − ξ²（若 |x|>ξ）。→ 小残差平方损失、大残差线性损失（抗肥尾/离群）。
- **式(6.32) R²_OOS（历史均值基准）**：1 − [ Σ_i (R_{i,t+1}−R̂_{i,t+1})² ] / [ Σ_i (R_{i,t+1}−R̄_{it})² ]，R̄_{it}=资产 i 到 t 期的历史均值。**判定**：<0 即不如历史均值。
- **式(6.33) R²_OOS（零基准，推荐用于个股）**：1 − [ Σ_i (R_{i,t+1}−R̂_{i,t+1})² ] / [ Σ_i R²_{i,t+1} ]。**判定阈值**：R²_OOS > 0.5% 即有价值；用历史均值基准会整体虚高约 3%。
- **式(6.34) Diebold–Mariano 统计量**：DM₁₂ = d̄₁₂ / σ̂_{d̄₁₂}，其中 d₁₂=(1/N)·Σ_{i=1}^N [ (e^(1)_{i,t+1})² − (e^(2)_{i,t+1})² ]，e^(1)/e^(2)=两模型对股 i 的预测误差。**判定**：DM₁₂ 越大→模型1越差；越小→模型1越好。
- **式(6.35) 隐性因子模型**：R^e_{it}=β'_i λ_t+ε_{it}，真实因子不可观测（λ、β 均未知）。
- **稳健回归两型**：加权回归（赋样本点不同权重，如以当期股票数为权）/ 稳健损失（Huber，抗肥尾）。
- **惩罚回归三型**：Ridge=L2、LASSO=L1、Elastic Net=L1+L2（兼做特征筛选）。
- **集成两路**：boosting（顺序迭代、拟合残差、弱→强）/ bagging（并行、bootstrap、平均；典型=随机森林）。

## A股特殊性 / 实证结论

- **Gu et al.(2020) 60 年美股大实证（方法论范本）**：1957–2016；94 公司特征 × (8 宏观变量+1) + 74 行业 = **920 特征**；比较 **13 个模型**——6 线性(全特征 OLS、仅规模/账面市值比/动量 3 特征 OLS、PLS、PCR、Elastic Net、GLM) + 2 树(随机森林、GBDT) + 5 神经网络(1–5 层隐藏层)；对 OLS/Elastic Net/GLM/GBDT 用 Huber 稳健估计。**结论**：
  - **全特征 OLS 极糟**（尤其对大盘指数）；**仅 3 特征 OLS / 加惩罚项的 Elastic Net 显著更好**——即"**约束**"是关键。
  - **表现最好 = 神经网络，尤其 3 层隐藏层**；R²_OOS 口径同结论。
  - **两两配对**：所有**约束线性 > 普通 OLS（显著）**；降维(PLS/PCR) ≈ 惩罚回归(无显著差异)；**树模型 > 线性(不显著)**；**NN > 线性(显著)，但 NN 对树的改进不显著**。
  - **特征四大类**（趋势类=动量/短期反转、流动性相关、风险测度、基本面）：**线性模型高度偏重趋势类，非线性模型更均衡关注多类**；总体**趋势类影响最显著**。
- **A股对照结论（关键差异）**：总体同样"**约束线性 > OLS，非线性 > 线性**"；非线性中 **DFN 与 XGBoost 表现非常出色**，集成学习可进一步提升。**最重要因子 = 交易摩擦类(流动性)相关因子，与美股（趋势类最显著）不同**。
- **A股制度的隐含连接**：本块未直接展开涨跌停 / T+1 / 停牌 / 注册制对 ML 建模的具体影响，但"**流动性因子在 A股 ML 中最重要**"这一实证差异，本身指向散户主导、交易摩擦更大的市场结构——做 A股 ML 因子时应**优先纳入并稳健检验流动性 / 换手 / 摩擦类特征**。[制度细节本块未展开]

## 踩坑对照（√252重叠年化 / 单调性判定 / 合成稀释尾部 / 其他）

- **√252 重叠收益年化**：**本块未直接涉及**。最接近的是 6.8.5"**预期最高夏普与波动率正相关 → 单次历史回测高估因子表现**"——同属夏普口径失真问题；书给的解药是**平减夏普(deflated Sharpe ratio)**。这与重叠收益下 √252 高估夏普是**互补**的两类"夏普被高估"现象，scorecard 两者都要防。
- **单调性判定**：**本块未涉及**（属分组排序检验，见前面章节）。
- **合成稀释尾部**：**本块未直接涉及**。但**混合预测 / 集成(取平均)** 在本块被书**正面**评价（平滑不同算法误差→更有效），与"合成会稀释暴露/损失尾部信息"是**张力关系**——区别在于：这里平均的是**多个模型对同一目标的预测**（降方差），而非把多个**因子暴露**线性叠加（稀释截面区分度）。落地时勿混淆两种"平均"。
- **其他（本块自带的 ML 专属坑，重中之重）**：
  1. **R²_OOS 基准陷阱**：个股层面用历史均值作基准会让所有模型 R²_OOS **虚高约 3%**（阈值仅 0.5%）→ 必须改用**零基准**[式 6.33]。
  2. **过拟合 + p-hacking**：单条价格路径反复训练 → Harvey et al.(2016) p-hacking；WFR 只避未来数据不根治。
  3. **CV 失效**：自相关 + 异方差使训练集泄漏到测试集 → 须保证训练/测试集**时间不交叠**。
  4. **样本过小**：≈700 月对 ML 太少 → 模拟多条更长价格路径补充。
  5. **黑箱**：用特征对输出的影响程度(Dixon–Halperin 2019)做事后归因，先建显著模型再解释。

## 对 scorecard / 回测框架的直接启示

1. **OOS R² 基准固定为"零"**（个股预测），并标注阈值 0.5%；严禁用历史均值基准——否则整体虚高 ~3%，足以误判。
2. **模型横向比较走 DM 检验**[式 6.34]，给出 t/标准误，而非肉眼比 R² 或夏普。
3. **ML 因子的样本外纪律**（写进回测框架硬约束）：训练/测试集**时间不交叠**、向前(WFR)但知其"只避未来数据"的局限、用**平减夏普**、必要时**模拟多条更长价格路径**做多场景。
4. **建模顺序"先约束线性、再上深度"**：实证反复显示约束线性(惩罚/降维) 即可大幅超 OLS，NN 对树的增量并不显著——scorecard 应记录"是否已穷尽约束线性"再判断深模型是否值得。呼应《ML in Asset Pricing》(Nagel 2021) "全特征联合 + L2 收缩优于逐因子显著性"、"缩放即先验"、"R²≠组合 Sharpe"。
5. **A股因子库优先级**：优先考察 + 稳健检验**流动性 / 交易摩擦类**特征（A股 ML 实证中最重要），与美股趋势类主导不同。
6. **PCA/IPCA 类因子加"可投资性"列**：IPCA 6 因子样本外 Sharpe 4.05 虽惊人，但受**卖空约束**、可投资性 / 换手存疑；scorecard 须含"卖空依赖 / 换手 / 小盘暴露"栏——直接呼应《Navigating the Factor Zoo》Ch12 的 **Fact 4（非线性仅边际增量、线性 IPCA 反而最稳健）** 与 **Fact 5（高换手、收益集中小盘/高成本股）**。
7. **可解释性栏入表**：用 permutation importance / Dixon–Halperin 的"特征→输出影响度"为每个 ML 因子做事后归因，作为黑箱模型上线前的必填项。
8. **另类数据归因警示**：本块未提，但与 Nagel Ch5"用当年不可得数据造的历史因子≈过度稀疏、伪 OOS 为正勿直接归因 alpha"一致——本项目用同花顺/东财概念热度、活跃度造历史因子时尤须警惕（见仓库 DATA_SOURCES）。
