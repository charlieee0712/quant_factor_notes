# Machine Learning in Asset Pricing · 第1章 引言 (Introduction) [PDF p.14–23 = 书 p.1–10]

> Stefan Nagel, *Machine Learning in Asset Pricing*, Princeton University Press, 2021 · 视觉精读(PDF 物理页 p.14–23，对应书内页 p.1–10) · 生成 2026-06-30

## 本块讲什么

全书引言。作者用一条主线把"机器学习 (ML)"引入资产定价：**资产定价的核心是预测问题 (prediction problems)，而真实世界里候选预测变量的数量极其庞大（高维）**，传统计量工具（尤其 OLS）在高维下失效；学术界长期靠"人为稀疏 (ad hoc sparsity)"回避这个问题，但这种回避既无理论依据、也可能正是现有资产定价模型表现不佳的根源。ML 正是为高维预测而生的工具箱，因此应被引入资产定价——但**不能"开箱即用 (off-the-shelf)"**，必须结合经济学先验做适配，Bayesian 统计是连接经济理论与 ML 的桥梁。本章后半部分交代术语对照、监督/非监督学习之分、本书的局限与各章组织。

本章是"为什么"之章，几乎不含具体方法，重在动机与世界观，奠定全书基调：**经济约束 (economic restrictions) 很重要；纯数据驱动的自动化预测"好得不像真的 (too good to be true)"。**

## 方法论 / 机理要点（重点·展开）

### 1. 预测问题是资产定价的核心（动机的根）
作者开宗明义：资产定价处处是预测。把这条贯穿到各个子领域，说明"高维预测"不是某个小众场景而是普遍结构：
- **给股票定价** → 必须预测公司未来现金流 (future cash flows)。
- **追求超额收益的交易策略** → 寻找能预测资产收益的信号 (signals)。
- **检验资产定价模型** → 找能预测资产间收益差、或能预测收益随时间可预测变动的预测变量。
- **信用风险模型** → 预测违约 (default)。
- **对冲与风险管理** → 预测资产收益的协同变动 (comovement / codependencies)。

机理含义：既然这些都是"在一组预测变量条件下求条件期望"的问题，它们本质上都是**监督学习 (supervised learning)** 问题——这为后文"ML 天然适配"埋下伏笔。

### 2. 候选预测变量的数量是"爆炸性"的（高维的来源）
作者用"信息供给史"论证维度之高，而非空谈：
- 仅"公司财报"这一窄切片，百年间数据量与变量数都呈爆炸式增长（见图 1.1）。1920 年代靠 Moody's 纸质手册 → 1990 年代 COMPUSTAT 电子库（每家公司数百~数千变量）→ 今天 SEC EDGAR 数据量级达 TB；**配合文本分析 (textual analysis)，单家公司理论上可构造上百万个变量。**
- 财报只是冰山一角：市场价格与成交历史是"巨量"数据；社交媒体可抽取情绪 (sentiment)；客户/员工在线评论 (online reviews) 可能含价值信息；还有无数其他数据源。

机理含义：维度 J（预测变量数）的天花板由"人能构造的特征数"决定，而这个数被技术进步推到了远超观测数 N 的程度。**关键不是"数据多"，而是"特征比样本多 (J 相对 N 大)"** —— 这才是统计上的麻烦。

### 3. 高维下 OLS 为何失效（坑·核心机理）
这是本章最硬的方法论点，必须抓牢：
- 以横截面股票收益预测为例：设 N = 5000 只股票（观测数），候选预测变量数 J 很容易超过 N。
- **当 J > N：OLS 没有唯一解**（设计矩阵不满秩，方程欠定）。
- **即便 J < N、但 J 不比 N 小很多：OLS 仍常给不出有用预测**——因为解释变量太多时，OLS **会去拟合噪声 (overfits noise)**，导致**样本内拟合好 (good in-sample fit)、样本外预测差 (poor out-of-sample forecasts)**。

这就是后续所有 ML 方法（正则化/收缩）要解决的根问题：**如何"约束/规训估计 (discipline the estimation)"，使训练出的模型在样本外有用。** 作者明确点出：ML 在实务中的成功，很大程度正源于发展出了有效的"规训估计"方法。

### 4. 实证资产定价中的"人为稀疏"及其弊（§1.1）
学术界过去如何回避高维？答案是**人为把模型降到低维**：
- **横截面收益预测**：每项研究只用少数公司特征 (firm characteristics) 做回归——整体看研究过很多特征，但单篇研究里预测变量数通常很小。
- **因子模型**：用极少因子概括投资机会集。例：Hou-Xue-Zhang (2015) 与 Fama-French (2015) 在市值加权市场组合超额收益之外，只加 3~4 个因子；这些因子是按规模、盈利、投资、账面市值比 (book-to-market) 等特征构造的组合。

作者的批判逻辑（弊端机理）：
- 在"潜在相关变量极多"的大背景下，只用少数因子，**实际上是对模型强加了极高的稀疏度——在成百上千候选因子里，把几乎所有因子的效应都设为零。**
- 强加极端稀疏能让传统统计方法"行为良好 (well behaved)"，但**这种稀疏是 ad hoc（拍脑袋、无依据）的**：提出低维模型的研究者只拿它对照过"全体可构造因子"的一个小子集，所以**我们根本不知道这些模型相对那一大堆被省略因子 (omitted factors) 的联合效应，到底漏掉了多少预测力。**
- 旁证：被视为"标准"的因子数量一直在随时间上升——Fama-French (1993) 三因子 → 四、五因子 → Barillas-Shanken (2018) 认为需要六个。一种解读是：**文献在缓慢地承认，确实存在相关的被省略因子。**

### 5. 理论资产定价中的"人为稀疏"（§1.2）—— 把批判推向更深
作者把矛头从"实证计量"指向"理论建模本身"，这是本章思想深度所在：
- 资产价格反映投资者对未来收益的预期；但**投资者的预期从何而来？** 真实投资者面对的预测环境，与实证研究者一样是高维的。
- **理性预期 (rational expectations) 假设比"预期理性"强得多**：模型里投资者**无需估计预测模型——他们已经知道它**，即完美知道任意相关预测变量与待预测变量之间的函数关系，并能据此算出条件期望。
- 常见辩护是："这代表投资者在稳定环境中学习足够久后达到的均衡"。但作者反驳：
  - 即便在低维环境，"学习已经收敛结束"的假设也存疑（引 Timmermann 1993；Lewellen-Shanken 2002；Collin-Dufresne-Johannes-Lochstoer 2017；Nagel-Xu 2019 —— 都论证了"投资者对数据生成过程参数的学习"对理解资产价格很重要）。
  - 在更现实的高维环境（投资者要从上千可观测变量里提取预测信息），"投资者早已学会"的说法**更不可信**。
- **核心主张（全书母题之一）：理论模型里的投资者，应当像研究资产价格数据的计量经济学家一样，在高维中挣扎。** 现有"投资者学习"模型却照样人为假设只用少数预测变量。作者大胆推测：**理论模型中投资者面对的预测难度，与现实预测难度之间的这种错配，可能正是现有理论资产定价模型实证表现不佳的原因。**

### 6. 机器学习是什么、为何适配（§1.3）
- ML 广义即"让计算机从数据中学习的算法"：喂训练数据 (training data) 学习 → 训练好的算法做预测。
- 图像识别为例（见图 1.2"热狗分类"）：特征 = 每个像素的数值色值，标签 = 类别；从已标注图像学到"像素 → 类别"的映射，训练完即可对未标注新图预测。其他例子：邮件反垃圾、基因表达预测肿瘤、自动驾驶解读传感器数据。
- **关键契合点**：这些 ML 应用里**特征数极大、常大于训练观测数**——正是 OLS 失效的设定；ML 的成功正源于发展出"规训估计、产出有用样本外预测"的有效方法。
- 作者对 ML 文献的定位很清醒：很多方法**对统计学并不新**，但 ML 社区**靠大量试验、聚焦"管用 (work)"而非估计量的理论性质**，攒出了一套在实际预测中被验证有效的方法。本书目标：综述资产定价引入这些工具的最初步骤、点明挑战、勾勒前路。
- ML 工具箱带来的机会：**实证上**让计量学家能纳入大量预测变量的联合效应而不必强加极端稀疏；**理论上**可把投资者建模为现实高维环境中的"机器学习者 (machine learners)"。

### 7. 全书反复出现的告诫：ML 不能开箱即用（最重要的世界观）
- 资产定价数据的性质，与科技、医学等领域**实质不同**；把 ML 工具原样照搬到资产定价**不一定管用**。
- 成功应用**往往需要适配 (adaptation)**；要做适配，就必须引入关于"数据生成环境"的**经济学先验知识 (prior economic knowledge)**。
- 直白结论：**"完全数据驱动、全自动做预测"这种 ML 联想"好得不像真的"。** 全书大量篇幅就在回答：**如何用经济推理 (economic reasoning) 让 ML 工具在资产定价中真正有效。**
- **桥梁 = Bayesian 统计**：它提供把先验知识纳入估计的框架，从而在资产定价的经济理论与 ML 之间架桥。全书多处用 Bayesian 视角去**解释 ML 方法**、并**提出受经济学驱动的改造**。（后文将揭示：正则化 = 对参数施加某种先验分布。）

### 8. 监督 vs 非监督学习（§1.5）—— 为何本书聚焦监督
- **监督学习 (supervised learning)**：基本就是回归方法。训练数据同时有特征 x_i 与标签 y_i，目标是找映射 y_i = f(x_i)。"监督"之喻：训练像有"老师 (teacher)"——学习者据 x_i 预测，老师揭示正确标签 y_i 告知对错，学习者据此调整对函数的估计；训练完即可对"只有特征、没有标签"的样本外数据预测标签。
- **非监督学习 (unsupervised learning)**：训练数据**只有特征、无标签**；目标是找数据的"压缩摘要 (compressed summary)"以抓住其本质性质。典型例：主成分分析 (PCA)——用更少的潜在因子近似一组变量，捕捉变量间大部分共同变动。
- **为何本书重监督**：资产定价的根本问题之一——"在一组预测变量条件下估计预期收益"——本质就是监督学习；现金流预测同理。非监督多扮演配角，如喂给监督算法前的**初始降维 (dimension-reduction) 步骤**。
- **重要细节（别误以为二分很硬）**：两者界线在资产定价里**没那么分明**——一些监督学习方法**自带类似非监督的降维元素**。（这是后续章节理解收缩/正则化的关键铺垫。）

### 9. 本书的四点局限（§1.6，定边界）
1. 这**主要是一本资产定价的书**，不是查 ML 最新进展之处；也不深入计算问题（计算很重要，但 ML 文献已覆盖好；**本书要填的空白是"ML 工具是否适配资产定价"的概念性问题**——这块此前关注少）。
2. **不是 ML-在-资产定价的穷尽综述**：只讨论一小部分方法，目标是点出机会与通用挑战，而非全面文献回顾。母题重申：**经济约束很重要**，要在挑工具、做设定选择时注入"有限剂量"的经济推理。
3. **聚焦横截面收益预测 (cross-sectional return prediction)**：其他领域（估值需预测基本面、信用风险需预测违约实现、风险管理需预测资产间协同依赖）ML 也有用且能处理非线性，但短书必取舍；此聚焦只是反映作者自身研究方向，盼洞见可外推。
4. **常抛开放问题而非给定论**：本书不提供"照菜谱做 (cookbook recipes)"，而是抛出未解问题以启发未来研究；末章 (Ch.6) 汇总开放研究议程。

### 10. 全书组织（§1.7，路线图）
- **Ch.2**：基础监督学习方法概览——预测连续变量的回归方法：ridge 回归、lasso、树与随机森林 (trees & random forests)、神经网络 (neural networks)；讲超参数 (hyperparameters) 及其**数据驱动调参 (tuning)**；超参数常控制**正则化 (regularization) 强度**；章末给出**正则化的 Bayesian 解释——正则化 = 对模型参数施加某种先验分布**，此解释贯穿后续章节用于注入经济推理。
- **Ch.3**：应用 ML 于资产定价的挑战——ML 算法所设想的数据性质 vs 资产定价数据的真实性质之间的关键差异；以"用各股自身价格历史做横截面收益预测"为贯穿实证例。核心警示：**资产定价信噪比 (signal-to-noise ratio) 极低**，在极灵活框架里"让数据自己说话"难有好结果，**必须对学习算法施加结构 (impose structure)**，把 ML 接到资产定价与组合选择的基本经济框架上，Bayesian 原则有助于此连接。
- **Ch.4**：沿此方向推进的具体做法——从均值-方差组合选择（等价于"把随机贴现因子 SDF 表示为资产收益线性组合"）出发，结合"市场中不太可能存在近似套利 (near-arbitrage) 机会"的观念，用 Bayesian 先验信念对风险-收益机会建模，从而**对收益预测问题施加受经济学驱动的约束**。所得估计量**类似 ML 常见的 elastic net**，但有重要差异：**它考虑了"预测误差的协方差 (prediction error covariances)"是组合风险-收益剖面的重要决定因素**。实证用一大批公司特征及其非线性变换（含特征两两交互 pairwise interactions）做预测变量；结论：**施加受经济学驱动的先验信念，对获得好的样本外预测表现很重要。**
- **Ch.5**：理论视角——若市场信息环境是高维的，理论模型里的投资者也应面对高维，故 ML 可作为投资者信念形成的建模蓝本。在一个"投资者须从历史数据学习现金流与一大批特征的关系"的简化环境里，投资者是 Bayesian 且把预测估计向"客观正确的先验"收缩。**均衡里收益样本外不可预测，但样本内强可预测**——因为事后做样本内分析的计量学家拥有实时投资者没有的"后见之明 (hindsight)"优势；**低维下这种隐性优势小，高维下很大。故样本内回归不适合用来推断风险溢价或投资者行为偏差。**（极重要的方法论警告，直击因子研究中样本内"显著"的可信度。）
- **Ch.6**：总结全书，提出 ML-在-资产定价的未来研究议程。

### 图 1.1（Figure 1.1, 书 p.2）— "公司财报：大数据 (Big data)"
**双对数 (log-log) 散点图**。横轴 = 变量数 (Number of variables)，范围 10^0–10^3；纵轴 = 数据量 (Megabytes)，范围 10^0–10^7。三个标注点呈"沿对角线向右上爆炸式增长"之势：

- **Compustat 1990**：约几十个变量（接近 10^1.5），约 10^3 MB（中部）。
- **SEC Edgar Database 2019**：约 10^3 个变量，约 10^6 MB（右上）。

**该图传达的内容**：百年间，公司财报这一"窄切片"的可用变量数增长约 2–3 个数量级、数据量增长约 5 个数量级，二者同步暴涨。它把抽象的"高维"问题坐实为一条可视的历史趋势，直观论证"候选预测变量数 J 已被技术进步推到极大"，从而引出"J 相对 N 太大 → OLS 失效"的统计困境。

### 图 1.2（Figure 1.2, 书 p.5）— "图像分类示例 (Image classification example)"
左侧 **特征 (Features) = 图像像素 X_i**：画成一个 3×3（带 "…" 表示更多）的像素网格，格内为数值色值（如第1行 25,25,25；第2行 25,38,29；第3行 27,39,39）。中间是待学映射 **y_i = f(X_i)?**。右侧 **标签 (Labels) y_i** 二分箭头：**0 = Not hot dog**，**1 = Hot dog**。

**该图传达的内容**：以最简化方式图解监督学习的结构——把高维数值特征（像素）通过待估函数 f 映射到一个标签。它让读者直观看到"特征 → 标签"的学习范式，也暗示像素这类特征"数目极大"，正对应"特征数 > 观测数"的高维设定。

### 关键"公式/定义"性陈述（本章无正式推导，提炼为命题）
- **高维失效条件**：当 J > N 时 OLS 无唯一解；当 J 接近 N 时 OLS 过拟合噪声 → 样本内好、样本外差。[符号：N = 观测数/股票数，J = 预测变量数]
- **监督学习目标**：求函数 f 使 y_i = f(x_i)，训练后对仅有 x、无 y 的样本外数据预测 y。
- **正则化的 Bayesian 等价（本章预告，Ch.2 展开）**：正则化 ≈ 对模型参数施加先验分布。
- **样本内 vs 样本外的"后见之明"命题（Ch.5 预告）**：均衡中收益样本外不可预测、样本内可预测，差距随维度上升而放大。

## 与因子研究 & 回测的关联（构造/检验/成本/L2/A股 视情况）

- **因子构造方向**：本书世界观直接支持"宽特征集 + 非线性 + 交互"的因子构造思路（Ch.4 用一大批公司特征 + 其非线性变换 + 两两交互作为预测变量），而非传统"少数几个手挑因子"。对自建因子库的启示：**不必、也不应过早人为稀疏化**——可大量构造候选特征，把"哪些有效"交给受约束的 ML 估计去决定。
- **检验设计（最关键的可操作教训）**：作者反复强调**样本外 (out-of-sample) 表现才是金标准**，样本内拟合好极易是过拟合噪声的假象。Ch.5 的"后见之明"命题更尖锐——**高维环境下样本内回归会系统性高估可预测性**，因此**用样本内显著性来论证因子有效、推断风险溢价是不可靠的**。回测中务必：严格区分样本内/外、用滚动/扩展窗口做真样本外、警惕高维下的"样本内强可预测"幻觉。
- **正则化/收缩是必需品**：因子数接近或超过样本数（A股横截面 ~5000 只、候选特征上千乃至更多，正是 J~N 或 J>N 的典型）时，OLS 不可用，须用 ridge/lasso/elastic net 等收缩方法；且收缩方向应由经济先验决定（如"无近似套利"→ 限制可达夏普比），而非纯数据驱动。
- **信噪比低是 A 股/全市场预测的共性难题**：Ch.3 点出资产定价信噪比极低，"让数据自己说话"易失败——必须给学习算法**施加结构**（经济约束 + Bayesian 先验）。这直接反对"无脑上深度模型/全自动 AutoML"的做法。
- **本书暂不涉及交易成本、Level-2/微结构**：本章明确聚焦横截面收益预测，未触及成本、L2 订单簿等；这些需另寻来源。（与本仓库已有的高频/L2、成本建模笔记互补，不重叠。）
