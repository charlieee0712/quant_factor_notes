## 本块讲什么

全书导论的前半部分。作者从"**流动性到底是什么**"这一根本问题切入，层层递进地搭出全书的概念骨架：

## 方法论 / 机理要点（重点·展开）

### 1. 流动性的三层定义：从一句话到一条曲线

**定性定义**：一项资产若"容易买、容易卖"则为 *liquid*（流动）。看似简单，但无法量化。

**为什么流动性重要——liquidity risk 的来源**：投资者按某价格给资产估值、想买入持有几个月再卖出，他必须量化**流动性风险**，即"真正建仓时要付多少钱"。作者点出四条致损机理：
- **时间漂移**：在市场上凑齐所需数量可能要好几天，期间价格可能朝不利方向移动；
- **信息泄露 / 被抢跑**：潜在卖方可能掌握与他相同的信息，或通过观察 orderbook 动态推断出价格该涨，于是若买方不够"隐蔽(stealthy)"，对手方会抬价卖给他——这正是 **market impact(市场冲击)**；
- **记忆效应**：等他想卖时，市场会不会"记得"他买过这么多，只肯给不利价格。

→ 结论：流动性不是单点报价问题，而是**带数量维度、带时间维度、带信息博弈的成本问题**。

**第一层量化代理：bid-ask spread**（最优买价与最优卖价之间的距离）。短视角下可作流动性代理，但**致命缺陷**是没有体现这些价位上"能成交多少量"。

**第二层量化代理：round-trip cost（往返成本）**——"立即买入再立即卖出"的净亏损（见图 1）。比 spread 好，因为它把"吃掉多少量"算进去了。但作者强调关键一句：**它不是一个数，而是一条曲线**——对不同需求数量分别计算，就得到"数量 → 单位成本"的映射。**这就是本书对流动性的核心刻画：流动性 = 一条曲线，不是一个标量。**

### 2. 大单切片与最优交易理论（slicing / optimal trading）

当所需数量在公开报价 / 电子 orderbook 里**不能瞬时拿到**时，投资者必须把大单拆成**切片(slices)**，分散到**不同时间**与**不同交易场所/对手方**。

如何拆才最优？要同时权衡两股相反的力：
- **市场风险(market risk)**：拖得越久，越暴露于价格随机漂移；
- **市场流动性 / 冲击(market liquidity & impact)**：抢得越快，冲击成本越高。

对这一权衡做数学优化 = **最优交易理论(optimal trading theory)**（详见第 3 章 3.4 节）。作者提醒：这类优化**能嵌入**市场冲击模型，但它本身**不告诉你该用哪个冲击模型**——模型选择是另一回事。

### 3. 市场冲击的弹性、衰减与同步性（resilience / decay / synchronization）—— 本书最核心机理

**弹性(resilience)** 是冲击的关键属性，定义性问题是：*"如果我在半小时内消耗了某资产的流动性、因冲击推动了价格，需要等多久价格才会回来？"*

三个递进的机理判断：

- **衰减(decay)不对称**：大买单造成的冲击，在**上涨市**与**下跌市**中的衰减并不相同。冲击不是一个对称的弹簧。

- **微观解释 = 同步性(synchronization)**：从微观看，冲击是否持久取决于你的买单与他人订单的"同步程度"。
  - 若你下大买单时，恰逢众多参与者也在买 → 冲击是**永久(permanent)**的；
  - 若同期多数人在卖 → 你的大买单冲击几乎**不可见(invisible)**，被对手流抵消掉。

- **如何测量冲击**：单次观测无意义，必须把冲击在**足够多的市场情形上做平均**，让各自特定的上下文相互抵消，才能显露出冲击的**内在价值与幅度(intrinsic value and amplitude)**（第 3 章 3.2 节讲同步效应与冲击测量）。

**冲击是价格形成过程 PFP(Price Formation Process)的主要因子**，判据极简洁：
> 与当前市场共识**不一致**的买卖压力 → 只产生**临时(temporary)**冲击；
> 与参与者观点**一致**的压力 → 没人来反向推回价格 → 冲击**永久**。

由此推出一个深刻直觉：**市场上观察到的价格震荡，本质是买卖双方的临时失衡；理论上若投资者更同步，这些震荡本可被消除。**

### 4. 做市商机理（market-maker）—— 把临时冲击变成利润，并由此"闭环"出流动性观

**做市商如何赚钱**：他们从"临时冲击"中套利——**先以低价买入早到的卖方，稍后再以高价卖给后到的买方**。这一行为**抹平了源自临时冲击的无意义价格震荡**（起到平滑器作用）。

**做市商为何仍有风险**：他们无法预知在"卖方到达"与"买方到达"之间是否突发新闻事件导致价格跳变——这就是**库存风险 / 市场风险**。微观结构理论 [O'Hara, 1998] 解释了这层**市场风险 ↔ 做市商 bid-ask spread**的关系：spread 是做市商承担库存与新闻风险的补偿。

**"闭环(the loop is now almost closed)"论证**：如果我们接受做市商的存在，那么：
- 大部分临时价格震荡会被压缩为一个**与该资产内在风险(intrinsic risk)相关的 bid-ask spread**；
- 于是流动性变得**与资产基本价值一致**，**不再是纯内生(endogenous)量**。

**但作者立刻指出这幅理想图景有两个漏洞**（这是本块思想最锋利处）：
- **漏洞一：流动性有上限**。某些资产的流动性**不可能超过其市场风险所设定的阈值**。"所有资产都极度流动的市场"是一个 *chimera(空想)*——接近 [Grossman and Stiglitz, 1980] 描述的"空想式有效市场"：在最低时间尺度上，**市场动态必须保留足够的无效率，才能奖励那些改进价格形成过程信息含量的参与者**。（完全有效 → 无人有动力做信息工作 → 矛盾。）
- **漏洞二：摩擦(frictions)**。众所周知有些套利**因摩擦而从未被实施**。那么：如果摩擦也阻止了做市商高效地"剥头皮(scalp)"价格震荡呢？

### 5. 从"摩擦"到"监管"再到"碎片化与 HFT"的因果链

漏洞二在 2000 年后引起监管注意。识别出的**摩擦成本**：交易所的**垄断**导致高费用、低服务质量。于是：
- 美国 **Reg NMS**(约 2005) 与 欧洲 **MiFID**(约 2007) 出台，核心逻辑是：**在交易场所间引入竞争**以降低显性与隐性摩擦成本 → 让做市商更高效 → 全面提升股票市场流动性。

**结果出乎意料**：流动性的性质本身变了——变成一个**高度碎片化(fragmented)**的系统，反而把做市商的效率打上问号。叠加金融危机对流动性的冲击，催生了新角色 **HFT(高频交易者)**：他们**像做市商一样行事，但大多没有做市义务(obligations)**，模糊了市场结构各层的传统角色。MiFID 2(2018 年生效) 即为修补这些意外后果而设计。

**全书三大问题**（导论给出的路线图）：
1. 如何**定量描述**一个碎片化市场？（第 1 章）
2. 如何**理解**这种市场各项特征之间的关系？（第 2 章）
3. 如何在这种环境下**优化交易**？（第 3 章）

### 6. HFT 作为"流动性桥"与碎片化的代价

- **HFT 是碎片化的代价**：不可能在让多个交易场所相互竞争的同时，又不需要有人去搭建跨场所的**高频流动性桥(liquidity bridges)**。其潜在负外部性需被审视。
- **核心问题**：市场参与者与整体市场结构，应当愿意付出多少代价来支撑这些高频流动性桥？
- **调节 HFT 活跃度的杠杆之一 = tick size(最小报价单位)**（第 1 章 1.3 节）。
- **跨市场套利机理**：因为必须保证"某处的买价不高于别处的卖价"（否则存在跨场所套利），套利者(HFT)会**吃掉两价之差的一半**，同时"改善"了那些只接入少数场所的参与者的信息水平。这类套利越多，参与者就能越"盲目地(blindly)"把单子发往任意场所——等于把信息搜寻**外包**给了 HFT 并为此"服务"付费。由于 HFT 在机制上**必然加剧碎片化**，其活跃度可用市场碎片化程度来监测（见 §11 的 FEI）。

### 7. PFP 与信息：分享信息的两难

PFP 主要由**信息**驱动。从单个投资者视角看存在张力：
- **分享信息最坏**：会恶化自己的市场冲击、更容易被**逆向选择(adversely selected)**；
- **利用他人信息最好**：在别人卖时买、别人买时卖，对自己的交易过程有利。

→ **时机(timing)与最优调度(scheduling)至关重要**（第 3 章 3.2、3.4 节）。美国有**合并行情带(consolidated tape)**来组织信息共享、让参与者能做同步决策、强化 PFP；欧洲缺这一机制，作者认为**合并的盘后行情带(consolidated post-trade tape)** 是个好选项，能把碎片化反过来用于增强 PFP 的稳健性。

### 8. RFQ vs CLOB：两种交易机制的本质差异（跨资产类别的关键）

股票是最电子化的市场；期货也电子化；美国期权部分电子化。但**固收市场**的标准自动化方式**不是限价订单簿(CLOB, Centralized Limit Orderbook)，而是 RFQ(Request For Quotes, 询价)**：交易者向多个做市商(dealer)发消息表达对某标的的兴趣 → 后者电子化回传"报价(quotes)"(双边价与量) → 交易者再挑一个 dealer 成交。

由此引出**两机制的本质对照**（方法论上极重要）：

- **结构**：RFQ 本质**双边(bilateral)**；CLOB 本质**多边(multilateral)**。
- **信息不对称**：RFQ 中两方信息不对称——交易者握有**基本面信息(fundamental information)**，做市商握有**订单流信息(flow information)**(该客户惯常需求 + 她其他客户的当前流)。（脚注：数据挖掘/AI 可用于做市商的多尺度流分析；见附录 A.4 Kyle's model 讲统计估计在 dealer 端的作用。）
- **成本暴露**：RFQ 中交易者更暴露于**机会成本(opportunity cost)**，做市商更暴露于**逆向选择成本(adverse selection cost)**；CLOB 中每个人**两种成本都暴露**，取决于他用限价单还是市价单。
- **last look / conditional orders**：做市商为防逆向选择喜用**最后一看(last look)**或条件单——本质是"**标准化的拒绝成交方式**"。RFQ 中广泛使用，CLOB 上罕见(块交易系统除外)。它引入又一层不对称(除非双方都能用)。
- **stalled quotes(失效报价) ≈ vanishing liquidity(流动性消失)**：
  - RFQ 端：交易者接受了一个**已不再有效**的报价，dealer 只能告诉他"太晚了"；
  - CLOB 端：交易者发可成交限价单去打他看到的最优买价，但在单子到达前，该价位已被别的成交吃掉、或挂单者撤单；
  - 二者本质相同，但 **stalled quotes 比 vanishing liquidity 更频繁** → RFQ 系统里"所见"比 CLOB 更不可靠("所见≠所得")。

**演化判断**：固收若被监管推向更电子化，会否最终走向 CLOB？2016 巴塞罗那 Fixed Income Leaders Summit 圆桌上，大行与投资者认为会演化出 RFQ 的电子化变体（all-to-all RFQ、RFS=Requests For Stream），而非直接跳到 CLOB——因为改变习惯与软件的成本太高，业界更可能先用"半成品"(多个 RFQ 两两同步拼出近似 CLOB 的视图)，多年后再切换。

### 9. 衍生品交易（derivatives）

- **期货**可像股票一样交易，但**盘后机制(margin calls 等)不同**。
- **期权**在美国可电子化交易(限价+市价单，多边平台)，也走 OTC；BATS 等另类场所也开期权场。欧洲期权难以引入跨场所竞争的主因是 **CCP(中央对手方)碎片化**——常常无法把"A 场所买、B 场所卖同一期权"轧差为零；缺乏 CCP 互操作性就难有衍生品交易竞争。EMIR 监管方向是**放松互操作性**。(脚注：BATS 已与 CBOE 合并。)

### 10. 微观结构的定义与交易者分类

- **微观结构(microstructure)定义**：市场中不同交易者**分担其职责与角色的方式**。规则由监管者定；在规则下每个交易者**最大化其效用(风险 vs 回报的平衡)**。
- 股票市场(欧美)交易者通常两分 [Harris, 2002]：
  - **agents(代理人)**：经纪商 + 买方交易员；
  - **principals(自营方)**：功利型交易者、投机者、做市商(dealers)、"噪声(noise)"交易者。
- **流动性 = f(交易规则, 交易者类型, 标的资产属性[行业/风险水平])**。因此**监管变化、公司治理决策(改变资本结构)、技术性决策(拆股/合股)都会改变一只股票的流动性**——流动性是制度与结构的内生产物。

## 关键公式 · 图表 · 定义（图表转述）

### 公式直觉：round-trip 单位成本

对需求数量 Q：
- 买入 Q 要沿 ask 侧向上"吃"多个价位，得到平均买价 `P_buy(Q)`(随 Q 上升)；
- 卖出 Q 要沿 bid 侧向下吃，得到平均卖价 `P_sell(Q)`(随 Q 下降)；
- **单位往返成本** `c(Q) = P_buy(Q) − P_sell(Q) > 0`，关于 Q **单调递增**；
- 极限 `Q→0` 时 `c(Q) →` 盘口报价价差(touch / quoted spread)。

机理：量越大 → 越深入订单簿 → 买更贵、卖更便宜 → 单位成本越高。这是大单要切片的根本原因。

### 图 1（书 p.2）：Crédit Agricole 的 orderbook 与典型 round-trip 曲线（2012-12-28 15:41 CET）

- **上图 = 某瞬时 orderbook 快照**。横轴 = 价格(约 6.070–6.095 欧元)，纵轴 = 数量(−20000 ~ +20000)。
  - 正值柱 = **买方(bid)侧**挂单，集中在约 6.073–6.083，量在最优价附近最高(峰值约 17000 股于 ~6.074–6.075)，越远离盘口量越小；
  - 负值柱 = **卖方(ask)侧**挂单，集中在约 6.084–6.093，最深处约 −16000 股(于 ~6.087)；
  - 中间 6.083–6.084 的空档即**买卖价差(spread)**所在。
  - 转述要点：**订单簿是离散、阶梯状、两侧不对称的**；可成交量随价格离盘口越远而起伏分布，这正是"流动性是曲线"的原始数据来源。

- **下图 = 由该订单簿算出的 round-trip 成本曲线**。横轴 = 交易数量(0–60000 股)，纵轴 = 每股平均(买价−卖价)成本(约 0.002–0.012 欧元)。
  - 曲线**单调上升且凹(concave)**：小量端从 ≈0.002(即盘口价差)起步，随数量增加先**陡升**后**趋缓**，到 60000 股时约 0.012；
  - 转述要点：**单位成本随规模上升**(吃得越深越贵)；凹形说明边际单位成本增速放缓，但总成本仍超线性增长。**流动性=这条曲线，而非 spread 这一个数。**

### 图 2（书 p.6）：碎片化市场微观结构示意图

- **自上而下分层的订单流向图**：
  - 顶层：**Investors(投资者)** 与 **HFT**；
  - 中间层 **Intermediaries(中介)**：OTC、**SOR(Smart Order Router, 智能订单路由)**、Crossing engine(撮合引擎)、Dark Pools(暗池)；
  - 底层场所群：两簇各含 **Visible book(明池)/Dark pool(暗池)/Mid point(中点)**，外加 **Primary(主板)、Hub、MTF(多边交易设施)** 及若干交易场所(图中多个 "TF" 节点)。
- **箭头机理**：投资者把单子发给中介(OTC/SOR/Crossing engine)；SOR 再把单子分发到多个明池/暗池/中点；HFT 横跨各场所与暗池(虚线/粗箭头)，充当**跨场所流动性桥**；各场所经 Hub 互联形成环路。
- 转述要点：**与"投资者→交易所"的单一链路相比，碎片化后路由层级骤增、HFT 嵌入为关键连接者**——这张图直观说明了"角色被模糊"和"为何需要 HFT 桥接"。

- 多条曲线：AEX、BEL20、CAC40、DAX、FTSE 60、SWISS LEADER PR INDEX。
- 形态：各指数 2008 年初从 0–10% 起步，**2008 年急升(第一段快斜率)**，**2009 年起转为缓升(第二段慢斜率)**，到 2011–2012 在约 **60%–80%** 平台震荡。
- **FEI 定义与用法**：受物理学**熵(entropy, 衡量环境异质性)** 启发构造(第 1 章 1.1.3 节)。指数越高(上限 100%)→ 越能"盲目"地把单子发往任一场所；越低(下限 0%)→ 越需要**谨慎拆单(split)**。即 FEI 高 = 跨场所价格越一致、套利桥越完善。
- 转述要点：**碎片化的"两段式"演进**——制度推动下的快速碎片化，随后进入有效但仍需精细路由的稳态。

### 表 1.1（书 p.15–16）：电子交易 40 年全球时间线（Chronology）

- **排版编码**(传达"变化类型")：等宽字体=**技术变化**；粗体=**新进入者(增加碎片化)**；斜体=**合并(减少碎片化)**。最左列标注当年**处理器主频与算力(MIPS, 百万指令/秒)**，按摩尔定律递增——把市场演化与算力演化并置，是本表的巧思。
- **关键里程碑（转述脉络，非清单堆砌）**：
  - **技术铺垫期**：1970s 美国 NYSE 的 **DOT**(电子路由订单到交易池)；1980s 指数套利与程序化交易自动化；1986 法国 **CAC** 全计算机撮合引擎；1990s 美国 ECN；1992 **FIX 协议**(国际实时信息交换)。
  - **制度奠基期**：2000s 美国价格**小数化(decimalization)**；2001 CESR 成立(后被 ESMA 取代)；2004 EU **MiFID** 指令；2005 美国 **Reg NMS**、**BATS** 成立。
  - **碎片化爆发期**：2006 NYSE Arca、NYSE-Euronext 合并；2007 **Chi-X**(阿姆斯特丹/伦敦/巴黎)——且 2007.08 单一参与者占 Chi-X 阿姆斯特丹成交的 80%(HFT 集中度信号)；2008 **Turquoise**、BATS Europe、Xetra Mid-Point；2009 **tick war**(BATS 与 Turquoise 竞相缩小 tick)、Flash orders、首个 Systematic Internalizer。
  - **风险与反思期**：2010-05-06 **Flash Crash**；2010 起 SEC/CFTC 调查、EC 启动 MiFID II 咨询；2011 多起交易所**宕机(outages)**(意大利、伦敦、Chi-X、NYSE Euronext) + 印度首批批准 SOR。
  - **整合期(合并主导)**：LSE-Turquoise、BATS-Chi-X、ICE 收购 NYSE Euronext(2013)、NYSE 与 Euronext 拆分(2014)；2016 ESMA 发布 MiFID II 技术标准；2017 CBOE 收购 BATS、SEC 批准 **IEX** 为全国性交易所；**2018-01 MiFID II 计划实施**。
- 转述要点：**技术(算力↑)→ 新进入者(碎片化↑)→ 危机暴露脆弱性 → 合并(碎片化↓)+ 监管收紧**，呈"扩张—收敛"的钟摆，且近年事件与合并明显增多。

### MiFID 2 关键设计（导论预告）

- **暗池上限(cap)**：单一暗池 ≤ 某标的成交流动性的 **4%**；全部暗池合计 ≤ 一标的成交的 **8%**(按年度执行：过去 12 个月内越线则后 6 个月禁交易该标的)。
- **两种盘前透明度豁免**：**LIS(Large in Scale)豁免**(超大单可免盘前透明度并参与形价) 与 **imported price 豁免**(小单引用别处可见价格则免盘前透明)。MiFID 1 暗池用后者；MiFID 2 的 cap 正是针对后者 → 想做暗交易者将转向用 LIS 豁免的新池(block discovery / block trading / size discovery)。
- **tick size 制度化**：tick 很可能设为**价格 × 日均成交笔数的函数**，以同时兼顾**离散化效应(因价格)** 与**流动性效应(因成交笔数)**。
- 交易者面临选择：继续暗交易(接受更大块) vs 回到明池连续交易 → 可能导致**流动性二分(bifurcation)**，需要把块交易与连续交易**智能组合**的算法(超出传统"流动性搜寻算法<2h、中小单"，转向 **IS=Implementation Shortfall / PoV=Percentage of Volume** 这类更长周期算法)。

## 与因子研究 & 回测的关联（构造/检验/成本/L2/A股）

- **成本建模(回测必备)**：本块的核心方法论——"**流动性是 round-trip 成本曲线 c(Q)，不是固定 bid-ask spread**"——直接否定了回测里"按固定 bps 滑点扣费"的粗糙做法。正确做法是用 L2 订单簿重建 c(Q)：买入沿 ask 累加、卖出沿 bid 累加，得到随委托量上升的单位成本曲线；大单要按曲线扣**超线性总成本**，否则系统性低估策略容量(capacity)与高估容量内收益。
- **冲击 = 临时 + 永久两段**：永久冲击(与共识同向)进入持仓成本且不可回收；临时冲击(与共识反向)会随**弹性(resilience)** 衰减回补。回测/TCA 中应把两者分开估计，并注意**衰减在涨/跌市不对称**。对因子换手率高的策略，永久冲击是真实 alpha 的主要侵蚀者。
- **同步性 = 拥挤度(crowding)的微观对应**：作者"买单与他人同步→永久冲击"的机理，正是因子拥挤/同质化交易导致冲击放大的微观解释。可据此构造**拥挤度/同步性监控指标**(如同向大单簇集、成交流不平衡)，并用于动态调节下单速度。
- **测量冲击需跨情形平均**：与因子检验同理——单笔执行的冲击充满噪声，必须在大量执行样本上聚合、让特定上下文相互抵消，才能估出**内在冲击幅度**；这等价于因子研究里的"多样本求期望、控制 idiosyncratic 噪声"。
- **A股映射与差异**：A股是 CLOB 连续竞价(非 RFQ)，但同样有 vanishing liquidity(撤单/抢成交)；本块的 round-trip 曲线、切片、冲击/弹性框架完全适用。差异点：A股 **T+1、涨跌停、最小变动价位(tick)固定**、无真正暗池/做市商连续报价(科创板/做市另议) → 弹性与冲击形态不同，回测需用本地 L2 重估 c(Q) 而非套用欧美参数。
- **数据需求**：复现本块方法需**逐档 L2 快照 + 逐笔成交/撤单**(本仓库另类/高频数据见 DATA_SOURCES.md)，才能重建订单簿深度、计算 c(Q)、估计 resilience 衰减曲线。
